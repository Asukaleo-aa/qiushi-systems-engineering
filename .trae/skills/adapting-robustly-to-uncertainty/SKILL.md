# 在不确定中自适应寻优

## 描述

在信息不完备时放弃一次性完美设计，将方案设计为能在运行中感知、试探、评价、修正的动态过程。提供三条自适应路径：自寻优（通过试探找到最优）、自镇定（通过切换找到稳定态）、模型参考（向着已知"对"的样子调整）。

## 使用场景

### 何时使用

- 不了解全貌，需要边做边调
- 运行一段时间后性能下降，需要自校准
- 环境变化太快，固定策略无法应对
- 不知道最优工作点在哪，需要探索
- 新领域没有参考方案，需要试探前进
- 已知条件不充分但必须开工

### 何时不使用

- 环境完全确定且已知（直接设计最优方案）
- 没有在线反馈能力（无观测则无法自适应）
- 决策必须一次性做出且不可逆转
- 试探成本高于执行错误成本（直接保守执行）
- 问题已完全在模型内（先用约束优化求解）

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| uncertainty_type | enum | `input_uncertain` / `plant_uncertain` / `reliability_uncertain` / `drift` |
| can_probe | boolean | 是否能施加试探并观测响应 |
| probe_cost | enum | `low` / `medium` / `high` |
| has_reference_model | boolean | 是否知道"对"的样子 |
| safety_constraints | list | 不可逾越的约束条件 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| strategy | enum | `self_optimizing` / `self_stabilizing` / `model_reference` / `combined` |
| probe_rules | object | 试探幅度、方向逻辑、观测指标 |
| switch_conditions | list | 模式切换触发条件 |
| safety_boundary | object | 回退条件、最大可接受损失 |
| adaptation_rate | object | 试探频率、步长规则、探索-利用平衡 |
| learning_curve | string | 预期多久接近最优、中间阶段 |

## 指令

1. **诊断信息不完备的类型**：判断属于输入信号不确定、对象特性不确定、元件可靠性不确定、还是环境参数漂移。若为输入信号不确定，先走状态估计。

2. **选择自适应策略**：
   - 若不知道最优工作点且可施加试探 → 选择**自寻优**：施加小幅试探，观测响应方向，变好则同方向继续，变坏则反向或换维度，响应变小则减小步长。每次只变一个维度。
   - 若有多个备选模式且不确定哪个稳定 → 选择**自镇定**：定义可接受边界，运行当前模式，触碰边界则切换模式，边界内稳定则保持。
   - 若知道期望行为但不知道如何达成 → 选择**模型参考自适应**：定义参考模型（期望行为），并行运行实际方案与参考模型，偏差扩大则调整执行参数使其缩小。

3. **设计试探安全边界**：定义最大试探幅度（不造成不可逆伤害）、回退条件（何时放弃试探回到安全方案）、切换代价（切换成本高则用保守策略）。

4. **确定自适应速率**：环境变化快且试探成本低 → 大步试探、频繁更新；环境变化慢且试探成本高 → 小步试探、低频更新；初期信息少 → 快速试探（探索为主），后期信息多 → 精细微调（利用为主）。

5. **输出自适应策略报告**：包含不完备信息诊断、选用策略及理由、期望行为定义、试探/切换规则、安全边界、自适应速率、学习曲线预期。

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 试探导致系统振荡或发散 | 立即回退到已知安全方案；减小试探步长；检查是否同时改变了多个维度 |
| 把噪声波动误判为系统变化 | 先用滤波平滑观测再决策；要求连续 N 次同方向变化才确认趋势 |
| 自适应速率与环境变化速率失配 | 监测自适应效果：若连续多轮无改善，切换速率档位（快→慢或慢→快） |
| 所有备选模式均不稳定 | 扩大搜索范围或降低稳定标准；若仍无解，回退到保守方案并上报 |
| 参考模型本身有误 | 用实践结果校验参考模型；若参考模型与实际持续矛盾，重新定义参考模型 |

## 示例

### 示例一：新领域策略选择

面对一个新领域的技术选型，不确定哪个方案最优。诊断：对象特性不确定，可施加试探，试探成本低。选择自寻优策略：对三个候选方案各做小规模原型验证，每次只变一个参数，观测性能指标变化方向，逐步逼近最优方案。安全边界：任何方案不得超出预算的 120%。

### 示例二：运行系统性能漂移

一个在线服务运行半年后响应时间逐渐增大。诊断：环境参数漂移，有参考模型（期望响应时间 < 200ms）。选择模型参考自适应：定义参考模型为"响应时间 < 200ms"，持续比较实际值与参考值，偏差超过 50ms 则触发参数调整（缓存策略、连接池大小等），调整方向根据偏差正负确定。
