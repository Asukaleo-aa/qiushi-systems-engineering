# Workflow 1：新项目启动流

**认知路径：** 感知 → 理解 → 执行

**适用场景：** 从零开始面对一个新任务，目标已知但路径未知，资源有限。

**触发信号：** 新项目、新领域、MVP、从零开始、不知道从哪里下手

## 工作流

```
investigation-first   →   first-principles   →   contradiction   →   spark-prairie-fire
   (感知：调查)             (理解：本质)           (理解：矛盾)          (执行：启动)
                                  │
                          state-estimation ──→
                          (感知：推断隐藏状态)
```

## 步骤详解

### Step 1：感知——摸清现状

调用 `investigating-before-deciding`（调查一手事实）。如数据噪声大、状态不可直接观测，并行调用 `estimating-hidden-states`（从观测推断状态）。

- 输出：事实清单 + 信息缺口 + 可靠信息源列表

### Step 2：理解——抓住本质

调用 `analyzing-from-first-principles`（剥离假设到不可再分，从基本元素重建理解）。

- 输入：Step 1 的事实清单
- 输出：问题本质的一句话定义 + 关键假设及其验证状态

### Step 3：理解——定位主要矛盾

调用 `analyzing-contradictions`（在本质理解基础上找核心张力）。

- 输入：Step 2 的本质定义
- 输出：⭐ 标注的主要矛盾 + 矛盾的主要方面 + 判定依据

### Step 4：执行——最小可行启动

调用 `sparking-prairie-fire`（在主要矛盾的方向上找最小可行切入点）。

- 输入：Step 3 的主要矛盾
- 输出：根据地选定 + 当前条件 vs 所需条件对照 + 发展路线

### Step 5（可选）：执行——长期布局

如任务短期无法完成，调用 `protracted-strategy`（划分防御/相持/反攻阶段）。

## 终止条件

- 最小可行方案已产出并验证
- 长期阶段的划分已明确
