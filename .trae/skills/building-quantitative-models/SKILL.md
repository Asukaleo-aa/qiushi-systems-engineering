# 构建定量模型

## 描述

从定性理解和经验性假设出发，建立可计算、可验证、可修正的定量模型。模型是对关键关系的抽象而非现实的完整复制——所有模型都是错的，但有些是有用的。

## 使用场景

### 何时使用

- 定性理解已充分，需要转化为可计算的数学/代码模型
- 需要对系统进行优化或预测
- 需要评估"如果...那么..."的场景
- 需要将经验判断固化为可检验的定量假设

### 何时不使用

- 纯定性分析不涉及数值计算
- 问题过于简单，可直接估算
- 无法获得建模所需的基本数据或参数
- 系统的定性理解还不充分（应先完成系统思维分析）

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| qualitative_understanding | object | 系统思维报告中的定性理解 |
| key_relationships | array\<string\> | 需要定量化的关键关系描述 |
| available_data | object | 可用的历史数据、参数来源 |
| modeling_goal | string | 建模目的（预测/优化/评估） |
| constraints | array\<string\> | 已知的约束条件 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| quantitative_hypotheses | array | 每个关键关系的定量假设与可检验预测 |
| variable_table | table | 完整的变量与参数表（名称/符号/单位/类型/来源） |
| model_specification | object | 模型定义（域/变量/方程/假设/参数/数值方法） |
| simplification_log | array | 每个简化假设的内容、理由、影响评估 |
| validation_results | object | 内部一致性检查、敏感性分析、历史数据对比 |
| limitations | array | 模型失效条件 |
| next_validation_plan | array | 需要进行的仿真或实验 |

## 指令

### 1. 建立经验性假设

对系统的每个关键关系，写出定量假设：定性描述 → 定量假设 → 可检验的预测。

示例：定性"温度升高加速反应" → 定量"k = A·exp(-Ea/RT), Ea≈50±10 kJ/mol" → 预测"T从300K升至310K时速率增约2倍"。

### 2. 识别变量与估计参数

完整列出模型涉及的所有变量，包含：变量名、符号、单位、类型（状态/输入/输出/参数）、来源/依据。

参数估计三来源：
- 已知常数：从权威来源直接获得
- 经验公式：基于已有研究的经验关系式
- 数据拟合：需通过实验或历史数据标定

### 3. 选择建模方法

根据系统特性选择建模方式：

| 系统特性 | 推荐方法 |
|---------|---------|
| 连续、可微、确定 | 微分方程（ODE/PDE） |
| 离散事件、随机 | 离散事件仿真 |
| 优化问题 | 运筹学（规划/对策/网络） |
| 不确定性显著 | 概率模型/随机过程 |
| 复杂系统涌现 | Agent-based/系统动力学 |
| 定性与定量混合 | 灰色系统/模糊逻辑 |

### 4. 构建模型

按以下结构定义模型：

- **域**：适用范围和边界条件
- **变量声明**：输入x、状态s、输出y、参数θ
- **核心关系**：状态方程、输出方程、约束条件，解释每项的物理/逻辑意义
- **简化假设**：每个假设必须显式列出，说明简化了什么、为什么可接受、对精度的影响
- **参数值**：值 ± 不确定度，附来源依据
- **数值方法**：积分方法与步长、优化方法与收敛判据、随机采样与样本数

### 5. 验证模型

执行多级验证：

1. **内部一致性**：量纲分析（方程两边量纲一致）、边界情况（极端参数下行为是否合理）、守恒律检查
2. **敏感性分析**：识别对结果影响最大的参数，决定标定优先级
3. **历史数据对比**（如有）：用已知数据检验预测能力，分析偏差与方差

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 关键参数无来源，无法估计 | 标注为待标定参数，给出合理范围和获取途径，触发调研补充 |
| 模型验证不通过（预测与数据严重偏离） | 回溯检查假设是否过度简化，逐步放宽简化条件重新构建 |
| 虚假精度（参数不确定度大但输出精度过高） | 降低输出精度表示，用不确定度传播量化结果的可信区间 |
| 模型对某参数极度敏感 | 优先标定该参数，若无法标定则标注为关键风险并给出参数变动时的结论变化范围 |
| 忽略反馈导致模型失真 | 识别缺失的反馈回路，补充到模型结构中，重新验证 |

## 示例

### 示例1：反应动力学建模

**输入**：定性理解"温度升高加速反应"，可用数据为3组温度-速率实验值，建模目标为预测。

**过程**：建立定量假设 k=A·exp(-Ea/RT)；识别变量（T为输入，k为输出，A和Ea为待标定参数）；选择ODE方法；用实验数据拟合参数得 Ea=48±5 kJ/mol；验证：量纲一致，预测与3组数据偏差<8%。

**输出**：模型规格、参数值与不确定度、验证结果、局限性（仅在280-350K范围内验证）。

### 示例2：系统效率建模

**输入**：定性理解"规模增大效率降低"，建模目标为评估N=50时的效率。

**过程**：建立假设 η=η₀/(1+αN)；参数α无直接来源，从3个已知数据点拟合得 α≈0.01±0.005；敏感性分析发现α对N=50时效率影响显著；标注为关键风险。

**输出**：模型规格、参数与不确定度、敏感性分析结果、建议优先获取更多数据点标定α。
