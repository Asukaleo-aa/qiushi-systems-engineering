# 形式化工具

## 描述

将 qiushi 方法论的定性分析翻译为控制论可计算结构，并提供 Python 执行路径。根据用户需求的量化程度，自动在定性、定量、混合三种模式间切换。

## 使用场景

### 何时使用

- 需要对多子系统进行耦合矩阵谱分析
- 需要验证当前方案是否落入假稳态
- 需要将定性描述构造为可计算模型
- 需要运行 FSM 场景测试或参数标定
- 需要查询哲学概念与控制论概念的映射关系
- 需要对残差进行分级诊断和路由

### 何时不使用

- 简单单步任务，不需要形式化分析
- 纯事实查询，没有系统结构需要分析
- 子 agent 执行的具体操作
- 用户没有提供任何系统描述或数据

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| analysis_target | "耦合矩阵谱分析" \| "FSM状态诊断" \| "参数标定" \| "残差分类" \| "概念映射" \| "auto" | 需要执行的形式化分析类型 |
| system_description | string | 用户对系统结构、子系统关系的描述 |
| available_data | list\<string\> | 可用的数值数据或观测值（可选） |
| noise_level | "低噪" \| "混合" \| "高噪" \| "未知" | 部署环境的噪声水平，用于参数标定（可选） |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| mode_used | "定性" \| "定量" \| "混合" | 实际使用的分析模式 |
| formalization_result | object | 形式化分析的结构化结果（内容随 analysis_target 变化） |
| data_gaps | list\<string\> | 缺失的数据项清单（混合/定性模式时必有） |
| conditional_predictions | list\<{if_data: string, then_result: string}\> | 条件推演（混合模式时输出） |

## 指令

1. **检查工具可用性**：检测 `qiushi-formalization` 是否已安装。可用则标注"定量模式可执行"；不可用则标注"仅定性模式"并提示安装方式。
2. **选择操作模式**：根据用户提供的系统描述和数据完整度选择模式——无数据选定性模式，有数据选定量模式，有描述但数据不完整选混合模式。
3. **执行对应工具**：根据 analysis_target 调用对应工具（详见下方工具清单），按选定模式输出结果。
4. **标注数据缺口**：混合/定性模式下，在结论中明确标注缺失数据，给出条件推演而非精确数值。
5. **输出格式化结果**：按各工具的标准输出格式呈现，定性推演结果必须标注"定性推演，未经数值验证"。

### 工具清单

| 工具 | 功能 | 定性模式做法 | 定量模式做法 |
|------|------|------------|------------|
| 耦合矩阵谱分析 | 识别主要矛盾及其性质 | 手动构造定性耦合矩阵，判断耦合强度和方向 | 构造数值矩阵，运行 CouplingMatrixAnalyzer |
| FSM 状态诊断 | 判定认知环位置和假稳态风险 | 查表判定三态，执行双通道门控检查 | 运行 main.py --scenario false-steady |
| 参数标定 | 查询门控参数推荐值 | 按噪声水平查表获取 σ_amp、N*、σ_dir* | 运行 run_scenario6.py 生成热力图和标定结果 |
| 残差分类 | 三级诊断框架路由 | 按幅度和方向一致性手动分类 | 同定性（此工具无独立定量脚本） |
| 概念映射 | 哲学⇄控制论翻译 | 查映射表 | 同定性（此工具为查表操作） |

### 耦合矩阵谱分析要点

- 提取子系统清单，构造 N×N 耦合矩阵
- 识别主要矛盾：耦合最强的子系统对
- 判定矛盾性质：资源竞争→对抗性；输入输出耦合→非对抗性；公共约束→视约束可松性；时序依赖→非对抗性
- 矛盾转化检测：比较前后两个状态的耦合矩阵，最强耦合对变化即矛盾转化

### FSM 状态诊断要点

- 三态判定：S₀ 大幅修正（未锚定/残差>20%）、S₁ 在线微调（已锚定/残差1.67%~20%）、S₂ 稳态（已锚定/残差<1.67%）
- 双通道门控：观测器通道（残差收敛）+ 元认知通道（坐标系锚定），观测器通过但元认知未通过→假稳态风险
- 超时风险：跟踪在当前态连续周期数，超限则建议回退

### 残差分类阈值

- Level 1 噪声：幅度 <5% 且方向散（dc<0.6）→ 门控统计过滤，不触发修正
- Level 2 漂移：幅度 5%~20% 且方向一致 → 降阶校正
- Level 3 结构：幅度 >20% 且方向一致 → 全重跑

### 概念映射表

| 哲学层概念 | 控制论等价物 |
|-----------|-------------|
| 实践验证 → 评估 | 卡尔曼滤波创新序列分析 |
| 主要矛盾 | 耦合矩阵的主特征值对 |
| 矛盾转化 | 耦合矩阵特征值排序重排 |
| 群众路线 → 集中 | 分布式卡尔曼滤波融合中心 |
| 第一性原理 → 剥离 | 区分本质约束与偶然约束 = 系统的最小实现 |
| 层次错位 | 在不同系统层次上优化同一目标 → 假稳态 |
| 批评与自我批评 | 人维度反馈环（元认知修正） |
| 反馈修正 | 系统维度反馈环（偏差修正） |

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 没有子系统描述就强行构造耦合矩阵 | 退回 investigating-before-deciding 摸清子系统边界后再构造 |
| 忘记标注数据缺口就用定量模式输出数值 | 缺失数据必须标注，给出"如果 X 则 Y"条件推演；定性推演结果标注"未经数值验证" |
| 把定性模式的推演结果当成精确计算 | 在结论中强制标注"定性推演，未经数值验证"；如需精确数值，提示安装 qiushi-formalization |
| qiushi-formalization 不可用但用户要求定量分析 | 切换到混合模式，构造结构并标注数据缺口，提示安装方式 |
| 耦合矩阵维度过大导致谱分析不可解释 | 先用 structuring-system-boundaries 做层次分解，在子系统层级分别构造矩阵 |

## 示例

**场景 1**：用户描述"研发团队有前端、后端、测试三个子系统，前端和后端争抢接口定义权，测试总是等两边定完才能介入"。

1. 选择工具：耦合矩阵谱分析
2. 构造定性耦合矩阵（3×3），前端↔后端耦合最强
3. 判定：主要矛盾 = 前端 vs 后端（资源竞争→对抗性），测试为次要
4. 建议：影子价格机制协调接口定义权

**场景 2**：用户声称"推荐系统已收敛，A/B 测试无显著差异"。

1. 选择工具：FSM 状态诊断
2. 双通道门控：观测器通道通过（残差小），元认知通道检查坐标系是否锚定
3. 发现：优化目标层次不一致（在页面级优化但业务目标在用户级）→ 假稳态风险
4. 建议：回退到 S₀ 重选坐标系
