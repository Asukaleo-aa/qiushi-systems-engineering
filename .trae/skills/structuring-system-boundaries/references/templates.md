# 构建系统边界与结构 — 模板

## 层次分解树模板

```
[系统名称]
├── 子系统A
│   ├── 组件A1
│   │   ├── 单元A1a
│   │   └── 单元A1b
│   └── 组件A2
├── 子系统B
│   ├── 组件B1
│   └── 组件B2
└── 子系统C
    └── 组件C1
```

```

├─ 功能：[一句话描述]
```

## 接口契约模板

```
接口：[接口名]
├─ 提供方：[组分名]
├─ 消费方：[组分名]
├─ 数据格式：[schema]
├─ 传输方式：[同步/异步/批量]
├─ 频率：[每秒/每步/按需]
├─ SLA：[延迟上限、吞吐量要求、可用性]
├─ 错误处理：[重试策略、降级方案、fallback]
└─ 版本：[接口版本号]
```

## 六维边界检查表

| 维度 | 检查问题 | 状态 |
|------|---------|------|
| 物理边界 | 所有物理实体是否被纳入？硬件/软件/人/环境？ | ✓/✗ |
| 逻辑边界 | 所有逻辑功能是否被覆盖？数据流是否完整？ | ✓/✗ |
| 时间边界 | 系统的生命周期阶段是否明确？（设计/开发/运行/退役） | ✓/✗ |
| 组织边界 | 哪些团队/人员负责哪些组分？权责是否清晰？ | ✓/✗ |
| 知识边界 | 系统的哪些部分已知（有模型支持）？哪些未知？ | ✓/✗ |
| 场景边界 | 系统的使用场景是什么？场景对系统施加了哪些不可转移的约束？ | ✓/✗ |

## 常见结构反模式

| 反模式 | 症状 | 修正方法 |
|--------|------|---------|
| 过度耦合 | 改一处动全身 | 解耦，定义清晰接口 |
| 层次混乱 | 高层管细节、低层做决策 | 恢复层次关系 |
| 遗漏组分 | 重要功能没有归属 | 重新遍历边界 |
| 接口不标准 | 组分间对不上 | 标准化接口 |
| 边界不清 | 无法判断谁负责什么 | 明确定义边界 |
