# 综合多源判断

## 描述

将多个信息源、分析方法和专家判断综合集成为统一判断。通过矛盾辨识、证据加权和定性-定量双向转换，把孤立的局部结论升华为有边界的综合决策依据。

## 使用场景

### 何时使用

- 不同来源给出矛盾结论，需要判定谁更接近实际
- 定性分析与定量结果指向不同方向，需要统一
- 决策前需要整合多方意见或多个agent的产出
- 从局部结论升华到整体判断

### 何时不使用

- 只有一个信息源或一个分析结果
- 所有来源指向一致结论且无矛盾
- 仍处于纯信息收集阶段，尚未到需要综合的时刻
- 简单任务不需要决策

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| sources | array\<Source\> | 待综合的信息源列表 |
| Source.conclusion | string | 该来源的结论摘要 |
| Source.type | enum | 定性 / 定量 |
| Source.confidence | enum | 高 / 中 / 低 |
| Source.system_level | string | 分析所在的系统层次 |
| Source.methodology | string | 得出结论的方法论基础 |
| decision_context | string | 当前决策的上下文与层次 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| integrated_conclusion | string | 一段话总结最终判断 |
| evidence_panorama | table | 各维度的结论、置信度、支持/反对证据、未决问题 |
| contradictions | array | 已识别矛盾及处理方式 |
| assumptions_and_bounds | object | 关键假设、有效范围、失效条件 |
| action_suggestions | array | 优先级排序的行动方案 |
| verification_needs | array | 需要进一步实践检验的判断 |

## 指令

### 1. 汇聚信息源

列出所有待综合的输入，为每项标注：系统层次、信息类型（定性/定量）、方法论基础、置信度（高/中/低）、适用范围。

若多个来源标注了不同系统层次，先做层次对齐判断：
- 同一层次内的结论冲突 → 真正的认知分歧，进入矛盾辨识
- 跨层次的"冲突" → 标注为"非对抗性分歧——层面错位"，先明确在哪个层次上做决策

### 2. 辨识矛盾

列出所有信息源之间的对立面，格式：`[来源A结论] vs [来源B结论]`。

判定哪对矛盾是主要矛盾（制约全局判断的分歧）。

区分矛盾性质：对抗性（方法论根本冲突，必须选边）或非对抗性（可调和的分歧）。

对主要矛盾标注主要方面（哪一方更接近实际）。

构建矛盾矩阵，为每对矛盾记录：分歧根源、矛盾性质、主要方面、处理策略（选边A / 选边B / 融合合理成分 / 保留分歧等待证据）。

关键纪律：
- 对抗性矛盾不得强行融合——必须在双方之间做出选择并说明理由
- 非对抗性矛盾运用"团结——批评——团结"方法，寻找分歧中的合理成分，统一为更高层次判断
- 证据不足无法判定矛盾性质时，返回调研补充后再继续

### 3. 加权整合证据

对每个关键判断点：列出支持和反对的证据，评估每项证据权重（基于方法论可靠性、数据质量、验证程度），计算综合倾向。

注意：
- 定量结果不等价于高质量——检查模型假设和简化
- 定性判断不意味着低价值——专家直觉往往整合了大量隐性知识
- 多数意见不代表正确——检查是否存在共同的系统性偏见

### 4. 定性-定量双向转换

定性→定量：将经验判断转化为可检验的定量假设（如"风险大"→"失败概率X%，损失Y"）。

定量→定性：将数值结果翻译为可理解的定性结论（如"p<0.01, d=0.8"→"有充分证据表明A优于B"）。

### 5. 产出综合判断

按 Output 结构输出综合集成报告，包含：集成结论、证据全景表、主要矛盾及处理、假设与界限、行动建议、后续验证需求。

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 信息源之间严重对抗且证据不足以选边 | 标注为"待定"，列出双方理由，返回调研补充事实后重新判定 |
| 关键参数/数据缺失导致无法加权 | 降级为定性综合，明确标注缺失项，触发调研补充 |
| 所有来源存在共同系统性偏见 | 引入外部视角或反事实推理，标注偏见的可能方向和影响 |
| 跨层次分歧无法对齐 | 分别在各层次产出独立判断，标注层次间关系，由决策者选择决策层次 |
| 综合结论对假设极度敏感 | 进行敏感性分析，列出假设变动时结论的变化范围 |

## 示例

### 示例1：技术方案选型的多源综合

**输入**：系统思维分析认为方案A架构更优（定性，高置信），仿真结果认为方案B性能更优（定量，中置信），领域专家认为方案A风险更低（定性，高置信）。

**过程**：识别主要矛盾——架构优势 vs 性能优势，属于非对抗性（不同维度）。加权整合：架构和风险维度倾向A，性能维度倾向B。定性→定量：将"风险更低"转化为"上线延迟概率降低约30%"。

**输出**：综合判断推荐方案A，条件是性能差距在可接受范围内；若性能差距超过阈值则需重新评估。

### 示例2：跨层次分歧处理

**输入**：物理层分析说Isp最高（定量，高置信），产品层分析说成本不划算（定量，高置信）。

**过程**：识别为层面错位——两者讨论的不是同一问题。在工程层面对齐：将Isp优势转化为成本影响，重新在同一层次上比较。

**输出**：标注为非对抗性分歧，在工程决策层次上综合判断：Isp优势带来的收益是否覆盖成本增加。
