# 递阶分解协调

## 描述

将整体任务分解为自治子系统（局部控制器），增设协调层只处理耦合约束，在递阶结构中实现局部自治与全局优化。子系统不互相窥探内部，协调层不干预子系统内部——只传递边界参数，子系统自行消化。

## 使用场景

### 何时使用

- 将大规模任务分解为多个子agent，但子agent的输出互相影响
- 多个子系统共享有限资源（算力、时间、上下文窗口）
- 局部优化方案汇总后出现冲突
- 需要"分而治之"但缺乏整合机制
- 在并行派发后子agent互不通讯导致全局不优

### 何时不使用

- 只有1-2个子任务且无耦合——直接执行即可
- 子系统间完全独立，不存在共享资源或公共约束
- 任务已被合理拆分且协调机制运转正常
- 需要强实时性的紧耦合系统——协调层的迭代通信可能太慢

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| subtasks | list[Subtask] | 待协调的子任务列表（≥3个） |
| couplings | list[Coupling] | 子系统间的耦合关系（资源竞争/输入输出依赖/公共约束/时序依赖） |
| global_objective | Objective | 全局优化目标 |
| resource_constraints | dict | 共享资源的上限约束 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| decomposition | dict[str, SubtaskSpec] | 各子系统的局部目标与约束集 |
| coordination_protocol | CoordinationStrategy | 协调策略（影子价格/目标值交互/约束松弛/里程碑同步） |
| convergence_result | ConvergenceReport | 迭代收敛状态与最终参数 |

## 指令

1. **辨识耦合变量**：列出所有子系统输入输出的交叉依赖，只记录跨子系统的外部依赖，子系统内部变量不纳入。
2. **设立子系统局部目标**：为每个子系统定义独立的目标函数和约束集，只包含该子系统能直接控制的变量；将原目标 J_i(x_i, x_j) 改写为局部目标 J_i(x_i; z_{j→i})，其中 z_{j→i} 是其他子系统传递来的耦合变量当前值。
3. **设计协调策略**：根据耦合类型选择协调机制——资源竞争用影子价格/拉格朗日乘子，输入输出耦合用目标值交互，公共约束用约束松弛，时序依赖用里程碑同步。
4. **执行协调迭代**：协调层广播初始参数 → 子系统独立求解并返回不可行/非最优反馈 → 协调层汇总反馈更新参数 → 重复至收敛或达到迭代次数上限。
5. **判断收敛**：满足以下任一条件即终止——各子系统反馈不再变化、全局指标改善低于阈值、达到预设迭代轮次上限、检测到振荡则切换降级策略（取最近可行解）。

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 协调层直接修改子系统内部状态 | 只传递边界参数，子系统自行消化；若已侵入则回退到上一轮参数重新广播 |
| 在子系统内部处理跨系统耦合 | 将耦合问题上抬一层交给协调层，子系统只处理局部约束 |
| 迭代不收敛（振荡或发散） | 取最近可行解作为降级方案；回退到 contradiction-analysis 重新判定主要矛盾是否根本冲突 |
| 一次性分解后不加迭代 | 强制至少执行2轮协调迭代，确认反馈稳定后才终止 |
| 用单目标优化处理多目标问题 | 对不可比较的指标转交 overall-planning 做权衡 |

## 示例

### 示例1：多agent共享算力池

三个子agent分别负责前端、后端、测试，共享GPU算力池。耦合变量为各agent的算力分配量。协调层用影子价格 λ_k 标定算力单价，各agent根据 λ_k 调整自身算力需求并反馈，协调层汇总后更新 λ_k，3轮迭代后收敛至最优分配方案。

### 示例2：模块间接口冲突

系统分解为数据层、业务层、展示层，业务层输出接口与展示层输入接口不匹配。协调层传递期望输出 z̄，业务层和展示层各自调整，2轮后接口对齐。
