# 估计隐状态

## 描述

从带噪声的观测中推断系统不可直接观测的内部状态。核心路径有两条：频域维纳滤波（需全貌统计，适合离线）和时域卡尔曼滤波（递推更新，适合在线）。共同本质是最优估计等于信号在观测空间上的正交投影。

## 使用场景

### 何时使用

- 系统状态不可直接观测，只能通过含噪声的输出间接推断
- 多个信息源互相矛盾，需要加权融合
- 数据流式到达，需要在线递推更新而非离线批处理
- 需要量化估计的不确定性，而非仅给出点估计
- 信号与噪声在某个域（时域/频域/空间域）中特征差异明显

### 何时不使用

- 系统状态完全可直接观测且无噪声
- 问题本质是逻辑推理而非统计推断
- 数据量极少，不足以支撑统计模型
- 信息可一次性全部获取且不需要流式更新
- 不确定性来自对手的策略性行为而非随机噪声（那是博弈问题）

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| signal_definition | string | 要估计的隐状态量 |
| observation_definition | string | 可观测的量（含噪声） |
| noise_model | string | 噪声类型/方差/相关性 |
| data_mode | enum: offline / streaming | 数据到达方式 |
| domain_hint | enum: time / frequency / spatial | 信号与噪声最易分离的域 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| point_estimate | string | 当前最佳估计值 |
| uncertainty | string | 误差方差或置信区间 |
| estimation_strategy | string | 选用的估计策略及理由 |
| information_gap | string | 最需要什么新信息来减少不确定性 |
| update_rule | string | 新信息到达时的修正规则 |

## 指令

1. **区分信号与噪声**：定义要估计的量（信号）、可观测的量（观测）、噪声的性质（类型/方差/相关性），判断信号与噪声在哪个域中最易分离。
2. **建立系统模型**：写出状态变量及其演化规则（状态方程），写出观测变量如何由信号和噪声组合（观测方程）。即使模型不精确也要建立——不准确的模型加滤波修正优于没有模型。
3. **选择估计策略**：根据数据模式选择——离线全量数据用维纳滤波；流式在线数据用卡尔曼递推；模型参数也未知用自适应滤波；信号非平稳用有限记忆滤波器；仅需检测信号存在用最优检测滤波器。
4. **执行递推更新**：对每个新观测，依次执行——预测下一状态和观测，获取实际观测，计算更新量（增益 × 残差），更新状态估计，更新不确定性。增益决定新观测权重：观测噪声大则增益小（信旧估计），模型不确定则增益大（快吸收新信息）。
5. **量化估计不确定性**：输出必须包含点估计和误差区间。宁可说"估计值是 X ± Y，置信度 Z%"，也不说"我认为就是 X"。
6. **声明操作边界**：定性推断直接推理即可；需要递推数值滤波时必须写代码执行，无法执行则退回定性模式并标注。

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 把观测直接当真实状态 | 区分"看到了什么"和"真实是什么"，观测 = 信号 + 噪声，必须经过滤波修正 |
| 所有信息源同等待遇 | 按噪声方差分配权重——低噪声源高权重，高噪声源低权重 |
| 攒够数据再做分析 | 在线场景必须用递推滤波，每条新数据在当前最佳估计基础上微调 |
| 模型不准就放弃建模 | 不准的模型 + 滤波修正 > 没有模型直接猜，先建近似模型再逐步修正 |
| 忽略信号非平稳性 | 若信号统计特性在变，切换为有限记忆或自适应方法 |
| 无法执行数值计算 | 退回定性模式，明确标注"以下为定性分析，非数值最优解" |

## 示例

**场景**：接手一个复杂项目，每周看到进度指标但指标含水分（噪声），需要估计真实进度。

1. **区分信号与噪声**：信号 = 真实完成率；观测 = 周报中的完成率（含夸大噪声）；噪声为正偏、方差约 15%。
2. **建立模型**：状态 = 真实完成率，自然衰减（延期风险）；观测 = 真实完成率 × 偏移因子 + 噪声。
3. **选择策略**：流式数据 → 卡尔曼递推。
4. **递推**：初始估计不确定，增益大，大幅修正；几周后估计稳定，增益降低，小幅微调。
5. **输出**：当前真实完成率估计 62% ± 8%，置信度 90%；最需要获取代码提交量作为辅助观测以降低不确定性。
