# 多表述等价变换

## 描述

同一系统可用多种数学语言等价描述。当在当前表述下问题难以处理时，通过等价变换切换到另一种表述，在目标表述中求解，再变换回来验证。等价变换保证变换前后的信息不增不减。

## 使用场景

### 何时使用

- 在某种数学框架下反复碰壁
- 微分方程难 → 试试代数方程；时域难 → 试试频域
- 随机分析复杂 → 试试希尔伯特空间几何化
- 需要可视化但当前是代数表述 → 试试相平面/频率图
- 数值计算不可行 → 试试解析变换

### 何时不使用

- 当前工具已经高效解决——不要为了"多角度"而多角度
- 只有一种已知方法且没有理由怀疑它
- 问题本质上是表述中立的（如逻辑推理、概念澄清）
- 已知目标表述的计算复杂度不比当前表述更低

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| current_representation | string | 当前使用的数学表述（时域/s域/频域/状态空间/泛函空间/几何/数值） |
| stuck_operation | string | 在当前表述下卡住的具体操作 |
| problem | object | 原始问题描述 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| target_representation | string | 切换到的目标表述 |
| transform_path | string | 变换路径（正变换 + 反变换） |
| solution_in_target | object | 目标表述中的解 |
| solution_in_original | object | 变换回原表述的解 |
| equivalence_verified | boolean | 反变换验证是否通过 |

## 指令

### 1. 明确当前表述

识别当前使用的数学语言：时域微分/差分方程、s域/z域传递函数、频域频率特性、状态空间向量微分方程、泛函空间算子/投影、几何相平面/参数空间、数值离散网格。

### 2. 列出候选等价表述与变换路径

| 源表述 → 目标表述 | 变换 | 使什么变简单 |
|-------------------|------|-------------|
| 时域微分方程 → s域 | 拉普拉斯变换 | 微分→乘法，卷积→乘积 |
| 时域差分方程 → z域 | Z变换 | 差分→乘法 |
| s域 → 频域 | 代入 s=iω | 稳定性→频率轨迹 |
| 高阶方程 → 一阶方程组 | 状态变量引入 | 数值积分，矩阵方法 |
| 状态空间 → 传递函数 | F(s)=H(sI-A)⁻¹B | 消除内部状态 |
| 时域随机 → 频域功率谱 | 傅里叶变换+维纳-辛钦 | 卷积→乘积 |
| 向量空间 → 正交基 | 谱分解/特征展开 | 耦合→解耦 |
| 偏微分方程 → 泛函空间 | 算子半群理论 | 集中参数系统类比 |
| 代数方程 → 几何 | 参数空间映射 | 稳定性区域可视化 |
| 连续系统 → 离散系统 | 采样+离散化 | 数字计算 |

### 3. 评估目标表述中核心操作的复杂度

只评估卡住的那个步骤在新表述下是否变简单：

| 卡住的步骤 | 推荐切换到 |
|-----------|----------|
| 解微分方程 | s域（变成代数方程） |
| 判断稳定性 | 频域（乃奎斯特/伯德图） |
| 分析随机响应 | 频域（功率谱密度） |
| 最优滤波 | 希尔伯特空间（正交投影） |
| 参数设计 | 参数空间几何（D-划分法） |
| 多变量耦合 | 特征空间（对角化） |
| 非线性周期振荡 | 描述函数法（谐波平衡） |
| 数值迭代收敛慢 | 变分表述（梯度信息） |
| 偏微分方程控制 | 泛函分析（算子半群） |

### 4. 在新表述中求解

直接在新表述中完成分析/设计，不需要保持与原表述的并行思考。

### 5. 变换回原表述并验证

将结果翻译回原始表述（通过反变换公式或对照表），在原始表述中验证解的正确性。变换不可逆 = 表述不等价——若翻译不回去，说明变换有信息损失，需换另一条路径。

### 操作边界

- **概念层面**：直接推理——"这个问题在频域看更简单，因为……"，不需要实际做傅里叶变换
- **需要实际变换**：必须写代码——FFT、拉普拉斯变换、状态空间转换需要数值计算
- **禁止**：声称做了变换并给出变换后的精确结果，但实际没有运行任何代码

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 变换后在新表述中从头重建整个问题 | 只变换卡住的那一步，其他保持不变 |
| 反变换不可逆（信息损失） | 换另一条变换路径；若所有路径都有损失，说明表述不完全等价，需补充约束 |
| 在多个表述间跳来跳去不落地 | 选定一个目标表述，解完再回；禁止并行试用多种 |
| 所有表述下都难 | 困难不在表述而在问题本身，回到矛盾分析确认主要矛盾 |
| 目标表述的变换计算量反而更大 | 放弃该目标表述，重新评估候选列表中的其他选项 |

## 示例

### 示例 1：时域微分方程 → s域

**问题**：RLC 电路的瞬态响应，时域微分方程求解繁琐。

1. 当前表述：时域二阶微分方程
2. 卡住操作：解微分方程 → 切换到 s域
3. 拉普拉斯变换后变成代数方程，直接求传递函数
4. 部分分式展开 → 反拉普拉斯变换回时域
5. 验证：时域解满足原微分方程 ✓

### 示例 2：状态空间 → 频域判断稳定性

**问题**：多变量反馈系统，状态空间中判断稳定性需要计算特征值。

1. 当前表述：状态空间 (A, B, C, D)
2. 卡住操作：判断稳定性 → 切换到频域
3. 求开环频率特性，画乃奎斯特图
4. 由乃奎斯特判据直接判断闭环稳定性
5. 验证：与特征值计算结果一致 ✓
