# 约束下优化

## 描述

在目标明确、行动空间有边界、每步选择改变可行空间的条件下，找出数学上可定义的最优策略。核心方法论线索：最优决策具有可继承的子结构——整体最优意味着每步最优，且最优解通常触碰约束边界而非停留在内部。

## 使用场景

### 何时使用

- 目标函数可量化（时间、成本、收益、误差等），需要最大化或最小化
- 控制变量受硬约束（预算上限、物理极限、逻辑限制）
- 阶段性决策中每步选择影响后续可行空间
- 资源有限需在多个竞争用途间分配
- 需要识别哪个约束是真正的瓶颈

### 何时不使用

- 目标本身模糊，尚未定义清楚"好"的标准
- 选项之间不存在可比较的共同度量
- 约束是软性的、谈判性的而非硬性限制
- 决策时间窗口极短，来不及形式化
- 不确定性的影响远大于优化空间的影响

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| objective | string | 目标函数（最小化/最大化什么量） |
| objective_type | enum: integral / terminal | 积分型（过程累加）还是终端型（只看结果） |
| control_variables | list<{name, range}> | 控制变量及其可行区间 |
| hard_constraints | list<string> | 不可违反的物理/逻辑限制 |
| constraint_type | enum: A-type / B-type / mixed | A型（分量独立受限）还是B型（总量受限） |
| system_dynamics | string | 状态变量受决策影响的演化规则 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| optimal_strategy | string | 最优决策的可执行行动规则 |
| bottleneck_constraint | string | 限制进一步优化的瓶颈约束及其影子价格 |
| sensitivity | string | 约束放松 10% 时目标改善幅度 |
| suboptimal_alternative | string | 最优不可行时的次优方案及代价 |
| problem_type_diagnosis | string | 问题结构类型诊断（线性/非线性、连续/离散等） |

## 指令

1. **定义目标与约束**：校验目标层次——当前目标是终端目标还是中间指标？若定义在错误层次，最优解可能在产品语境下无意义。写出目标函数（单一可比较量）、控制变量（每个的可行区间）、硬约束（区分物理不可逾越与惯例限制）、系统状态演化规则。
2. **判断问题结构类型**：线性系统加限制型约束 → 极大值原理（开关型控制）；线性系统加二次型指标 → 黎卡提方程（线性反馈）；非线性加一般约束 → 标准方程组加两点边值问题；离散阶段决策 → 动态规划；模型不确定 → 转交自适应策略。关键判断：控制变量受限时，最优解通常触碰边界而非取中间值。
3. **选择解法并求解**：连续控制问题用极大值原理——写出状态演化方程，引入伴随变量（影子价格），在每个瞬间选择使控制对目标的边际贡献最大化的行动。阶段性决策用动态规划——从最后阶段倒推，每阶段决策 = 立即收益 + 后续已算出的最优继续值，到初始状态时全局最优自然浮现。
4. **检验最优性**：验证每个时刻/阶段的决策在约束下确实取得极值；验证最优路径的任意后缀子路径对于其起点仍最优；检验最优解是否触碰约束边界——若所有约束松弛，检查目标函数形式。
5. **转化为可执行方案**：识别瓶颈约束并做灵敏度分析；评估最优解与次优解的距离（近则容错大，远则必须盯住）；确定执行中需监控的指标以确保不偏离最优轨线。
6. **声明操作边界**：定性策略分析直接推理；定量数值求解必须写代码执行，无法执行则退回定性模式并标注。

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 目标不清就开始优化 | 先定义可度量的目标函数，没有目标函数无从谈"最优" |
| 受限变量取中间值 | 受限变量在最优解中通常触碰边界（Bang-Bang），除非有严格凸理由取内点 |
| 正向枚举所有可能 | 阶段性决策用动态规划倒推，正向枚举是指数爆炸 |
| 忽略约束间耦合 | B型限制（总量受限）和A型限制（分量独立受限）最优解完全不同，必须区分 |
| 只做一次优化就不管 | 条件变化须重新求解，最优是相对于给定约束和目标的 |
| 无法执行数值求解 | 退回定性模式，标注"以下为定性分析，非数值最优解"，绝不假装求解了方程 |

## 示例

**场景**：5 人团队有 3 个项目排队，预算 15 人月，项目 B 合同要求至少 3 人月，问如何分配最大化总收益。

1. **定义目标与约束**：目标 = 最大化预期总收益；控制变量 = 每项目投入人月；约束 = 总人月 ≤ 15、B ≥ 3。
2. **判断结构**：离散选择 + 线性约束 → 背包型组合优化。
3. **求解**：B 投入 3（合同最低），A 投入 8（单位人月收益最高），C 投入 4（剩余资源投向次高收益）。
4. **检验**：每个项目的边际收益在分配点处递减顺序成立，总预算恰好用尽（触碰边界）。
5. **转化**：瓶颈约束 = 总人月；若预算增至 16.5，多出 1.5 人月全部投 C（边际收益次高）；次优方案 = B 投 4 人月，代价取决于收益函数。
