# 工程级扩展：结构化知识地图

> 当调查涉及陌生领域、需要多源验证、或需要在调研报告中区分已知/推断/假设时，启用此扩展模式。

## 确定调研范围

明确调研的边界和深度：

```
调研目的：[为什么要做这个调研？要回答什么核心问题？]
调研范围：[涉及哪些领域/学科？]
调研深度：[需要到什么粒度？原理级/方案级/实现细节级？]
时间约束：[调研需要多少时间？]
```

## 构建四层知识地图

按系统思维框架组织调研，挖掘以下层次：

- **哲学层**：该领域的基本假设、范式、世界观
- **理论层**：核心原理、基本规律、公认模型
- **方法层**：常用工具、标准流程、最佳实践
- **执行层**：实际案例、可操作方案、实现细节

## 多源信息采集与可靠性标记

| 信息源类型 | 示例 | 可靠性 | 获取方式 |
|-----------|------|--------|---------|
| 学术文献 | 论文、专著 | ★★★★ | 搜索、阅读 |
| 官方文档 | API文档、标准 | ★★★★ | 查阅 |
| 实践经验 | 案例、事故报告 | ★★★ | 收集 |
| 专家意见 | 论坛、访谈 | ★★-★★★★ | 引用 |
| 社区共识 | GitHub Issues、论坛 | ★★-★★★ | 搜索 |
| 推测/假设 | 理论推断 | ★-★★ | 推理 |

**强制多源交叉验证**：每个关键发现必须有至少 2 个独立来源确认。

## 信息分类与标记

对每一条关键信息标注：

| 标记 | 含义 |
|------|------|
| `[事实]` | 多个独立来源证实，可视为确定 |
| `[推断]` | 基于已知事实的合理推论，需验证 |
| `[假设]` | 当前接受但未经严格检验的前提 |
| `[存疑]` | 来源单一或不可靠，需进一步确认 |
| `[未知]` | 应该知道但目前完全不了解 |
| `[分歧]` | 不同来源给出不同结论，原因待查 |

## 知识缺口分析

```
知识缺口清单：
- [ ] 缺口1：[描述 + 为什么重要 + 影响什么决策]
- [ ] 缺口2：[描述 + 为什么重要 + 影响什么决策]

每个缺口评估：可以接受吗？
- Yes → 需要什么假设来填补？
- No → 需要什么行动？（实验？更多调研？）
```

## 结构化调研报告输出

当启用工程模式时，在基础调查结论之外额外输出：

```
## 领域调研报告

### 1. 关键发现（按四层知识地图组织）
### 2. 信息来源清单（表格：来源｜类型｜可靠性｜引用）
### 3. 知识缺口清单（表格：缺口｜重要性｜填补计划）
### 4. 矛盾与分歧（不同来源的矛盾结论 + 原因分析）
### 5. 调研结论
    - 已知/确定：[列表]
    - 合理推断：[列表]
    - 关键假设：[列表]
    - 尚不知：[列表]
```

## 两模式串联协议

当同时使用哲学模式和工程模式时：

1. 先用哲学模式出调查提纲（确保方向正确）
2. 再用工程模式按四层知识地图填充结构化内容（确保覆盖完整）
3. 两方结论对照：一致项确认为 `[事实]`；不一致项标注为 `[分歧]`，触发 `contradiction-analysis` 或 `meta-synthesis-engine`

## 原著依据

工程扩展模式来源于钱学森、宋健：
- 《论系统工程》第25章"社会科学研究的定量方法"
- 《论系统工程》第8章"用科学方法绘制国民经济现代化的蓝图"

哲学模式来源于毛泽东：
- 《反对本本主义》（1930年5月）："没有调查，没有发言权"、调查的七项注意
- 《改造我们的学习》（1941年）："实事求是"——"'实事'就是客观存在着的一切事物，'是'就是客观事物的内部联系，'求'就是我们去研究"
