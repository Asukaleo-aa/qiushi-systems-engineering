# 认知技能路由

## 描述

在每次顶层对话开始时，建立「先看事实，再下判断」的总原则，然后根据任务复杂度将后续工作路由到合适的认知技能。按认知环节逐环推进——感知→理解→建模→求解→执行→校验→修正，每个环节只加载当前需要的 skill。如果事实与预想冲突，修改的是判断，不是事实。

## 使用场景

### 何时使用

- 新的顶层对话开始，需要判断任务应走哪条认知路径
- 任务包含系统/复杂/建模/优化/不确定/整体等信号
- 轻量路由无法匹配，或匹配了 3+ 个 skill 需要串联
- 用户显式要求深度分析

### 何时不使用

- 被派遣执行单一具体任务的子 agent
- 简单问答、单步操作、事实查询（深度 0，直接行动）
- 宿主平台已在执行等价流程，只需吸收「实事求是」约束即可

## 输入输出

### Input

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| user_task | string | 是 | 用户提出的任务描述 |
| context | object | 否 | 当前对话上下文、工作区状态 |
| is_sub_agent | boolean | 是 | 是否为子 agent 执行单一任务 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| depth_level | integer | 深度等级（0-4） |
| active_rings | list[string] | 将展开的认知环节列表 |
| primary_skill | string | 主 skill 名称 |
| secondary_skills | list[string] | 辅助 skill 列表（可选） |
| routing_reason | string | 路由判断理由 |

## 指令

### 1. 判断是否需要路由

检查 `is_sub_agent`：若为 true，跳过此 skill。检查任务是否为简单问答或单步操作：若是，设 depth_level=0，直接行动。

### 2. 执行轻量路由

扫描任务关键词，匹配关键词→Skill 快速匹配表（见 `references/routing-table.md`）。若匹配到唯一 skill，设为 primary_skill，直接调用。若匹配到 3+ 个 skill 或关键词均不匹配，进入深度路由。

### 3. 执行深度路由（仅复杂任务）

定义系统层次（物理系统/工程系统/产品系统/社会-技术系统）。执行六级深度扫描（见 `references/depth-criteria.md`），确定 depth_level 和 active_rings。宣告深度等级和将展开的环节。

### 4. 按认知环推进

按感知→理解→建模→求解→执行→校验→修正的顺序逐环推进。每个环节只加载当前需要的 skill（见 `references/cognitive-ring-detail.md`）。维护状态机，每跨越一个环节时宣告当前位置和出口条件。

### 5. 遵守调度纪律

一次只选 1 个主 skill；确有必要再串联第二个。不为了形式完整而机械遍历全部七个认知环节。用户明确指示 > 宿主平台规则 > 本体系。

### 6. 闭环或终止

校验环节量化偏差，修正环节根据偏差层次决定回退到哪个环节。若连续两轮经过同一环节而没有获得新信息，宣告认知环停滞，终止循环。

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 关键词无法匹配任何 skill | 默认调用 `investigation-first` 先调查，再根据调查结果重新路由 |
| 多个 skill 同等匹配且无明确优先级 | 默认选择认知环中最靠前的环节对应的 skill（感知 > 理解 > 建模 > 求解 > 执行 > 校验 > 修正） |
| 认知环停滞（连续两轮无新信息） | 宣告停滞，输出当前已有结论和未解决的缺口，不再循环 |
| 深度路由判断模糊（介于两级之间） | 取较低深度等级，先轻量推进，遇阻再升级 |
| 下游 skill 调用失败或阻塞 | 回退到上一认知环节，检查是否遗漏了关键信息或假设 |

## 示例

### 示例 1：简单任务（深度 0）

**用户**：「帮我格式化这段 JSON」

**路由判断**：任务单一明确，depth_level=0，直接执行，不展开任何认知环。

### 示例 2：复杂任务（深度 3 — 全环展开）

**用户**：「我们的微服务架构越来越难维护，服务间调用关系混乱，每次改一个服务都会影响其他服务。帮我想想怎么改进。」

**路由判断**：关键词「系统」「复杂」→ 触发深度路由：
- 系统层次：产品系统
- depth_level=3（全环展开）
- 路径：感知（investigation-first）→ 理解（contradiction-analysis）→ 建模（systems-thinking-framework）→ 求解（constrained-optimization）→ 执行（engineering-orchestration）→ 校验（practice-cognition）
