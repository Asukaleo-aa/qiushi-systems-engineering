# 持久战略

## 描述

面对复杂长期任务，既不急于求成（速胜论），也不畏难放弃（悲观论），而是划分阶段、稳步推进。战略上保持信心，战术上每一步都严谨对待。在防御中求进攻，在相持中积优势，在反攻中抓转折。

## 使用场景

### 何时使用

- 面对明显需要长时间才能完成的复杂任务
- 当前处于劣势（资源不足、信息不够、能力暂时不足）
- 任务进行中遇到瓶颈，感到焦虑或想放弃
- 需要规划分阶段的推进策略
- 别人催促快速完成，但判断不可能速成

### 何时不使用

- 任务可在当前会话内完成——直接做完，不需分阶段战略
- 用户只需要一个具体步骤的执行结果
- 问题已有清晰解法，只缺执行——用 concentrating-effort-on-priority 集中完成
- 处于某一阶段的具体执行中，不需要重新评估战略

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| goal | string | 长期目标描述 |
| current_state | {resources, progress, weaknesses} | 当前我方条件 |
| task_conditions | {scale, obstacles, opportunities} | 任务条件评估 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| current_stage | enum | 当前阶段：strategic_defense / strategic_stalemate / strategic_counteroffensive |
| stage_assessment | {advantages, disadvantages, key_obstacle, core_task, forbidden_actions} | 阶段评估 |
| local_attack_point | string | 当前阶段可立刻集中力量拿下的局部进攻点 |
| stage_transition_condition | string | 转入下一阶段的具体条件 |
| checkpoint | string | 阶段检查点——完成后重新评估的里程碑 |

## 指令

1. **客观评估态势**：全面评估我方条件（资源、能力、进展、弱点）和任务条件（规模、障碍、机遇），既不夸大困难也不低估困难，实事求是。
2. **拒绝速胜论和悲观论**：速胜论（"马上搞定"）→纠正为承认需要时间、分阶段推进；悲观论（"做不到"）→纠正为困难是暂时的、坚持就能胜利。
3. **划分三阶段**：
   - **战略防御期**（探索与积累）：站稳脚跟，保存实力，积累认识；调用 investigation-first 建立基础认知；不急于求成，允许退让以保存力量；做能做的小事积累第一手经验。
   - **战略相持期**（最困难也最关键）：逐步积累优势，等待转折点；调用 concentrating-effort-on-priority 在每个具体问题上集中力量；保持耐心，持续推进不因缓慢而气馁。
   - **战略反攻期**（突破与收获）：利用积累的优势全面推进；条件成熟时果断出击；自信但不大意，识别转折点把握时机。
4. **在防御中求进攻**：即使整体处于困难阶段，也要在具体的局部问题上采取主动——战略上持久战，战术上求速决，用小胜利积累信心和优势。
5. **识别转折点**：持续观察条件变化——积累的认识是否足够？资源和能力是否到位？外部条件是否成熟？主要障碍是否已消除？条件成熟时果断转入全面推进。
6. **保持主观能动性**：在整个过程中保持积极态度和主动精神，不被暂时困难压倒，利用一切有利条件，在不利中创造有利。
7. **设置阶段检查点**：明确在完成具体里程碑后重新调用本技能评估阶段，判断是否需要转入下一阶段。

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 无法判断当前处于哪个阶段 | 默认归入战略防御期，先积累信息和资源，条件明确后再调整 |
| 战略防御期长期无法转入相持期 | 检查是否低估了障碍或高估了资源，调用 investigation-first 重新评估，必要时调整目标范围 |
| 相持期丧失信心想放弃 | 回顾已积累的小胜利清单，重新评估我方优势变化，确认转折点是否正在接近 |
| 误判转折点过早转入反攻 | 发现推进受阻时立即回退到相持期策略，不强行冒进，分析受阻原因 |
| 外部催促导致速胜论压力 | 输出阶段评估表说明当前阶段判断依据和跳过阶段的风险，坚持分阶段推进 |

## 示例

### 示例一：大型遗留系统重构

目标：将单体应用重构为微服务。当前：3人团队、对旧系统理解有限。

- 阶段判断：战略防御期（能力/资源暂时不足，需先积累）。
- 核心任务：理解旧系统核心业务流程。
- 禁止事项：不要急于拆分服务、不要同时改造多个模块。
- 局部进攻点：选择一个最独立的模块先行拆出（调用 concentrating-effort-on-priority）。
- 转入相持期条件：完成首个微服务独立部署且运行稳定。
- 检查点：首个微服务上线运行1周后重新评估。

### 示例二：学习复杂技术栈

目标：掌握 Kubernetes 生态。当前：仅有基础 Linux 经验。

- 阶段判断：战略防御期（知识缺口大，需先积累）。
- 核心任务：在本地跑通一个最小 K8s 集群并部署一个应用。
- 禁止事项：不要直接上生产环境、不要同时学多个高级特性。
- 局部进攻点：用 minikube 部署一个无状态应用。
- 转入相持期条件：能独立编写 Deployment/Service YAML 并理解其含义。
- 检查点：完成3个不同类型应用的部署后重新评估。
