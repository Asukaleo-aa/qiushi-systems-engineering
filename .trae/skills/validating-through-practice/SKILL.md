# 通过实践验证认识

## 描述

一切真知来源于实践。认识从实践中来，又必须回到实践中去验证和发展。这个循环永不终止，每一轮都比上一轮更接近真理。本技能负责将调查阶段形成的判断推进到验证阶段，在实践中检验认识、修正偏差、螺旋上升。

## 使用场景

### 何时使用

- 提出了方案或假说，需要验证其正确性
- 在学习新领域，需要从浅入深地理解
- 前一轮尝试失败，需要总结经验进入下一轮迭代
- 发现自己在纯粹"想"而没有"做"（教条主义倾向）
- 发现自己在纯粹"做"而没有"想"（经验主义倾向）
- 需要决定方案是否正确，却缺乏实践检验

### 何时不使用

- 任务是一次性输出，不需要迭代（如写一封邮件）
- 用户已有明确的验证结论，只需执行
- 当前处于调查阶段还未形成假说（先调用 investigation-first）
- 问题的正确答案可以直接查阅（文档、API 参考）

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| hypothesis | string | 待验证的方案、假说或判断 |
| current_stage | enum | `sensing` / `reasoning` / `validating` / `elevating` |
| validation_method | string | 可用的验证手段（运行、测试、原型、仿真等） |
| termination_condition | string | 本轮循环完成的判定条件 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| stage_declaration | string | 当前所处认识循环阶段的明确宣告 |
| practice_results | object | 实践中观测到的事实和数据 |
| hypothesis_verdict | enum | `confirmed` / `refuted` / `partially_confirmed` |
| lessons_learned | string | 本轮学到的内容 |
| next_stage | enum | 下一轮循环的起始阶段 |

## 指令

1. **宣告当前认识循环阶段**：每次进入必须先明确宣告——感性认识阶段（正在积累第一手材料）、理性认识形成阶段（正在从现象提炼规律）、实践验证阶段（正在检验假说/方案）、总结升华阶段（正在将验证结果升华为新认识）。

2. **投入实践，获取感性认识**：直接接触问题本身，亲自动手、亲自体验、亲自调查。不急于下结论，耐心观察，积累印象和数据。禁止过早给出结论。

3. **从感性认识飞跃到理性认识**：将散乱印象整理、分析、综合——从现象中提炼规律，从个别中归纳一般，形成假说、模型或方案。警惕教条主义：不套用现成理论模板，理性认识必须从具体实践中生长。警惕经验主义：不停留在零散经验，必须提升为系统理论。写出"我的假说是：……"，禁止直接实现未验证的假说。

4. **回到实践中验证**：将理论、方案、假说拿回实践中检验——制定验证计划、执行验证、观察结果是否符合预期。禁止把"感觉对"当作验证通过。不能因喜欢某方案就把失败解释成偶然，也不能因一次碰壁就否认全部经验。

5. **评估验证结果**：成功 → 认识基本正确，进入下一轮深化；失败 → 认识有误，回到第一步重新实践和认识；部分成功 → 认识部分正确，需要修正和补充。禁止用"我觉得对"或"理论上应该对"替代实践验证。

6. **螺旋上升，进入下一轮循环**：总结本轮经验教训，写出"本轮学到的是：……，下轮将……"，在新的认识基础上开展新的实践。禁止不总结就直接开始下一轮。永远不认为认识已经"完成"。

7. **在开始循环前明确终止条件**：写出"当 [具体条件] 时，我认为本轮实践认识循环完成。"

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 陷入教条主义——只引用理论不验证 | 立即停止理论推演，强制要求至少一次实践验证；用事实检验理论 |
| 陷入经验主义——只做不想 | 暂停执行，强制要求写出从经验中提炼的规律和假说 |
| 一次失败就放弃 | 回顾"一个正确的认识往往需要多次反复"；将失败视为认识过程的一部分，总结后继续 |
| 照搬别人的做法不分析自身条件 | 对比自身条件与参考案例的差异，针对差异调整方案后再验证 |
| 循环无法收敛——反复验证反复失败 | 检查假说是否在错误层次上；回退到 investigation-first 重新调查根本前提 |

## 示例

### 示例一：验证性能优化方案

提出假说"改用缓存可将响应时间降至 200ms 以下"。当前阶段：实践验证阶段。执行：在测试环境部署缓存方案，压测观测响应时间。结果：平均 180ms，但 P99 为 350ms。判定：部分确认——平均达标但尾部未达标。教训：优化方案对长尾请求无效。下轮：针对长尾场景设计专项方案。

### 示例二：新领域学习迭代

学习一个新框架，不确定最佳架构模式。当前阶段：感性认识阶段。执行：用最简单的模式实现一个原型，记录遇到的问题。提炼假说："该框架适合事件驱动模式"。验证：用事件驱动模式重写，对比开发效率和运行性能。结果：开发效率提升但调试困难。判定：部分确认。教训：该模式适合生产环境但需配套调试工具。下轮：研究调试工具链。
