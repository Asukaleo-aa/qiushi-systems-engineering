# 对偶互补分析

## 描述

每一个正向问题都存在一个对偶的反向问题。正向路径被卡住时，构造对偶系统、在对偶空间中求解、再将解翻译回原问题，往往获得新的数学结构或直观洞察。对偶不是"相同的另一个"，而是"互补的另一面"：状态↔协态、控制↔观测、向前时间↔向后伴随、代价↔收益、源↔汇。

## 使用场景

### 何时使用

- 正向求解某问题时遇到不可逾越的障碍
- 问题有天然的对称结构：输入/输出、原因/结果、状态/观测
- 需要从可观测数据推断不可测状态（状态估计/滤波问题）
- 需要在"控制成本"和"观测精度"之间做互换
- 优化问题中原始变量很多但对偶变量很少

### 何时不使用

- 问题只有一个明确的单向因果方向
- 不存在任何形式的对称/互换结构
- 正向方法已经高效解决，无需绕路
- 对偶变换的成本超过收益（简单问题）

## 输入输出

### Input

| 字段 | 类型 | 说明 |
|------|------|------|
| primal_problem | Problem | 原问题的数学表述（控制系统/优化问题/观测估计/因果链） |
| blockage | str | 正向路径受阻的具体描述 |
| symmetry_hints | list[str] | 可观察到的对称/互换结构线索 |

### Output

| 字段 | 类型 | 说明 |
|------|------|------|
| dual_system | Problem | 构造出的对偶系统 |
| dual_solution | Solution | 对偶空间中的求解结果 |
| duality_gap | GapType | 对偶间隙类型（强对偶/弱对偶/无直接数值对偶） |
| translated_solution | Solution | 翻译回原问题的解 |

## 指令

1. **识别原问题的数学结构**：将当前问题写成标准形式——控制系统写为 ẋ=Ax+Bu, y=Hx；优化问题写为 min f(x) s.t. g(x)≤0；观测量与状态写为 y=Hx+噪声；因果链写为 A→B→C→D。
2. **构造对偶系统**：按原问题类型选择对偶变换——能控性用矩阵转置+角色互换，状态方程用时间反向+转置，优化问题引入乘子构造拉格朗日对偶，根轨迹取倒数，因果链反转方向。判断对偶系统是否更简单（维数降低/约束线性化/有限维化）。
3. **评估对偶间隙**：确定对偶解与原问题解的一致性——凸优化+Slater条件或线性规划为强对偶（零间隙，可直接用对偶解）；非凸为弱对偶（有间隙，对偶解仅作参考界限）；定性互补则从对偶获取几何直观/结构理解。
4. **解对偶问题**：在对偶空间中求解，使用与原问题相同或更高效的数学工具。若需要数值解则必须写代码（cvxopt / scipy.optimize），不可仅声称结果。
5. **翻译回原问题**：将对偶解用原问题语言重新表述——对偶变量 λ* 对应原约束的影子价格，协态 ψ(t) 对应状态对指标的灵敏度，伴随系统输出对应原系统输入的"影响力"。若翻译不回去则返回第2步重新检查变换。

## 失败策略

| 失败场景 | 处理方式 |
|---------|---------|
| 认为对偶就是"反过来做一遍" | 对偶变换有明确数学规则，按原问题类型严格选择变换方式，不可随意反转 |
| 不检查对偶间隙就用对偶解替代原解 | 非凸问题有间隙，只能做参考界限；必须先评估 duality_gap 再决定是否直接使用 |
| 解了对偶但翻译不回去 | 每一步变换都保持等价映射关系；翻译失败说明对偶构造有误，返回第2步重新检查 |
| 声称对偶最优值但未实际求解 | 对偶思维（定性推理）与实际对偶求解（数值计算）必须区分；需要数值解时必须写代码 |
| 对偶问题并不比原问题简单 | 放弃对偶路径，回退到原问题用其他方法求解 |

## 示例

### 示例1：优化问题对偶

原问题：min f(x) s.t. g(x)≤0，变量维度100，约束5个。构造拉格朗日对偶后对偶变量仅5个，在对偶空间求解后得到 λ*，翻译回原问题得到5个约束的影子价格，为原问题提供灵敏度信息。强对偶成立，对偶解即为原问题最优解。

### 示例2：能控性→能观测性

原问题：判断系统 (A,B) 的能控性，矩阵维数高难以直接计算。构造对偶系统 (A^T,B^T)，判断其能观测性（计算更简便），由对偶原理知对偶系统能观测等价于原系统能控，间接得出结论。
