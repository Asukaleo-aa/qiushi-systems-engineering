# 综合集成引擎

> "处理复杂行为系统的定量方法学，是科学理论、经验和专家判断力的结合。这种定量方法学，是半经验半理论的。"
> —— 钱学森《论系统工程》增订版说明

## 核心原则

**孤立的分析没有决策价值。综合集成是钱学森系统工程方法论的最高体现——将定性认识、定量模型和经验判断三者融合为一个统一的判断。**

本 skill 是全部系统工程的"大脑"——它接受所有局部产出的信息，经过矛盾辨识、证据加权、逻辑整合，产出可供决策的综合判断。

---

## 不适用场景

以下情况**不需要**调用本 skill：
- 只有一个信息源或一个分析结果
- 所有来源指向一致结论且没有矛盾
- 当前处于纯信息收集阶段，尚未到需要综合的时刻
- 简单任务且不需要决策

## 方法论流程

### 第一阶段：信息汇聚

> 钱学森强调："提出经验性假设（猜想或判断），是建立复杂行为系统数学模型的出发点。"

**先列出所有需要综合的输入：**

| 来源 | 系统层次 | 类型 | 结论摘要 | 置信度 |
|------|---------|------|---------|--------|
| 系统思维分析 | 物理/工程/产品/社会-技术 | 定性 | ... | 高/中/低 |
| 定量模型结果 | 物理/工程/产品/社会-技术 | 定量 | ... | 高/中/低 |
| 仿真验证结果 | 物理/工程/产品/社会-技术 | 定量 | ... | 高/中/低 |
| 领域知识/专家经验 | 物理/工程/产品/社会-技术 | 定性 | ... | 高/中/低 |
| 外部文献/数据 | 物理/工程/产品/社会-技术 | 定量/定性 | ... | 高/中/低 |

**每项来源标注：**
- 系统层次：分析是在哪个层次上进行的？（物理系统 / 工程系统 / 产品系统 / 社会-技术系统。见 `systems-thinking-framework` 第零步）
- 信息类型：定性 vs 定量
- 方法论基础：基于什么方法得出的
- 置信度评估：高（有严格验证）/ 中（有部分验证）/ 低（基于推测）
- 适用范围：在什么条件下有效

**关键纪律——跨层次标注：** 如果多个输入来源标注了不同的系统层次，在进入矛盾辨识之前必须先做层次对齐判断：
- 同一层次内的结论冲突 → 真正的认知分歧，进入常规矛盾辨识
- 跨层次的"冲突" → 可能是层面错位而非真分歧（例：物理层分析说 Isp 最高，产品层分析说成本不划算。两者都对，但讨论的不是同一个问题）。在这种情况下，标注为"非对抗性分歧——层面错位"，需要先明确"我们到底在哪个层次上做决策"

### 第二阶段：矛盾辨识（联合 `contradiction-analysis`）

> 钱学森指出不同的人对系统科学的理解"各一词，莫衷一是"，强调要正视分歧。
>
> 本阶段直接集成毛泽东《矛盾论》的矛盾分析法（`contradiction-analysis`），形成"定性矛盾分析 + 定量证据加权"的双重辨识机制。

**步骤 2a：调用 `contradiction-analysis` 产出矛盾分析表**

在进入本 skill 的证据加权整合之前，先对信息源之间的冲突执行完整的矛盾分析流程：
- 列出所有信息源之间的对立面（格式：`[来源A的结论] vs [来源B的结论]`）
- 判定哪对矛盾是主要矛盾（制约全局判断的分歧）
- 区分矛盾性质：`对抗性`（方法论根本冲突，必须选边）或 `非对抗性`（可调和的分歧）
- 对主要矛盾标注主要方面（哪一方更接近实际）

**步骤 2b：将矛盾分析结果转化为矛盾矩阵**

```
矛盾矩阵：
  来源A vs 来源B：在 [X] 方面存在分歧
  - A 说：...
  - B 说：...
  - 分歧根源：[方法论不同 / 假设不同 / 数据不同 / 范围不同]
  - 矛盾性质：[对抗性 / 非对抗性]（由 contradiction-analysis 判定）
  - 主要方面：[A/B]（矛盾的主要方面决定问题的性质）
  - 处理策略：[选边A / 选边B / 融合A和B的合理成分 / 保留分歧等待更多证据]
```

**关键纪律：**
- 当矛盾属于对抗性时，不得强行融合——必须在 A 和 B 之间做出选择并说明理由
- 当矛盾属于非对抗性时，运用"团结——批评——团结"方法：寻找分歧中的合理成分，将其统一为更高层次的判断
- 如果矛盾分析中发现证据不足无法判定性质，返回 `investigation-first`（基础模式补事实，工程扩展模式补结构化覆盖）补充调研后再继续

### 第三阶段：证据加权整合

> "这种定量方法学，是半经验半理论的。"

**对不同来源的结论进行加权整合：**

对于每个关键判断点：
1. 列出支持和反对的证据
2. 对每项证据评估权重（基于方法论可靠性、数据质量、验证程度）
3. 计算综合倾向：
   ```
   P(结论A) ≈ Σ(支持证据权重) / Σ(所有证据权重)
   ```

**尤其注意：**
- 定量结果不等价于高质量——检查模型的假设和简化
- 定性判断不意味着低价值——经验丰富的专家直觉往往整合了大量隐性知识
- 多数意见不代表正确——检查是否所有来源有共同的系统性偏见

### 第四阶段：定性-定量转换

> "软科学是定性与定量相结合的系统科学。" —— 钱学森

**双向转换机制：**

1. **定性 → 定量**：将经验判断转化为可检验的定量假设
   - 例如："这个方案风险大" → "失败概率估计为X%，损失为Y"

2. **定量 → 定性**：将数值结果翻译为可理解的定性结论
   - 例如："p<0.01且效果量d=0.8" → "有充分证据表明方案A优于方案B"

### 第五阶段：综合判断产出

**输出格式——综合集成报告：**

```
## 综合集成报告

### 1. 集成结论
[一段话总结最终判断]

### 2. 证据全景
| 判断维度 | 结论 | 置信度 | 支持证据 | 反对证据 | 未决问题 |
|---------|------|--------|---------|---------|---------|
| 维度1   | ...  | 高/中/低 | ...     | ...     | ...     |

### 3. 主要矛盾及处理
- 矛盾1：[描述 + 性质 + 如何处理]
- 矛盾2：[描述 + 性质 + 如何处理]

### 4. 假设与界限
- 本结论基于以下关键假设：[列表]
- 本结论在以下范围内有效：[范围]
- 以下情况本结论可能失效：[条件]

### 5. 行动建议
优先级排序的行动方案（含理由和预计风险）

### 6. 后续验证需求
哪些判断需要进一步用实践检验？（触发 practice-cognition 循环）
```

## 与其他 skill 的关系

- **上游**：接收来自所有 analysis-skills 和 method-skills 的产出
- **下游**：综合结论传递给 `engineering-orchestration` 用于执行、传递给 `feedback-and-revision-loop` 用于持续修正
- **协作**：可以与 `contradiction-analysis`（矛盾分析）协同工作，深入处理复杂矛盾
- **循环**：如果发现关键信息缺失，返回 `investigation-first`（工程扩展模式）补充调研
- **工程控制论方法集成**：当信息源涉及复杂系统的分解-综合问题时，可调用以下技能进行深度处理：
  - `hierarchical-decomposition-coordination`：将多来源分析转化为多层协调结构
  - `duality-complementarity-analysis`：利用对偶视角发现矛盾的新解法
  - `perturbation-progressive-method`：对定量结果进行渐进精化
  - `multi-representation-equivalence-transform`：将定性判断转化为可计算的等价表述

## 黄金法则

1. **定性定量不可偏废** —— "半经验半理论"是精髓
2. **有矛盾才有进步** —— 正视分歧，不要强行统一
3. **证据有权重** —— 不是所有证据同等可信
4. **结论必须有边界** —— 说清楚"在什么条件下成立"
5. **假设必须透明** —— 藏在结论背后的假设比结论本身更需要审查
