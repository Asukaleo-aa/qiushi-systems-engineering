## 工具 1：耦合矩阵谱分析（矛盾形式化）

### 定性模式

当用户描述多个子系统及其相互影响时：

1. **提取子系统清单**：从描述中识别出 N 个独立子系统
2. **构造定性耦合矩阵**：对每对 (i,j)，判断耦合强度（0=无耦合，1=弱，2=中，3=强）和方向
3. **识别主要矛盾**：找到耦合最强的子系统对
4. **判定矛盾性质**：
   - 资源竞争（两个子系统争同一个预算/算力/人力）→ 对抗性
   - 输入输出耦合（A 输出 = B 输入）→ 非对抗性
   - 公共约束（被同一外部条件限制）→ 视约束可松性
   - 时序依赖（B 必须在 A 之后）→ 非对抗性

5. **矛盾转化检测**：比较前后两个状态的耦合矩阵——如果最强耦合对发生了变化 → 矛盾转化已发生

### 定量模式

```bash
cd ~/Desktop/qiushi-formalization/fsm-prototype
python3 -c "
import numpy as np
from fsm_core import CouplingMatrixAnalyzer

# 用户提供的耦合矩阵
A = np.array([[...], [...], ...])
ca = CouplingMatrixAnalyzer(A)
result = ca.analyze()
print('特征值:', result['eigenvalues'])
print('主导子系统对:', result['dominant_pair'])
print('排序:', result['sort_order'])
"
```

### 输出格式

```
耦合矩阵 A (N×N):
  [行 i, 列 j] = 子系统 j 对 i 的影响强度

谱分析结果:
  主导特征值:  λ₁ = X.XX
  主要矛盾:    子系统 {i} ↔ 子系统 {j}
  矛盾性质:    [对抗性/非对抗性]
  协调策略:    [影子价格/目标值交互/约束松弛/里程碑同步]

矛盾转化检测:
  上一轮主导对: ({i_old}, {j_old})
  本轮主导对:   ({i_new}, {j_new})
  结论:         [未转化 / I 类参数漂移 / II 类结构转化]
```

## 工具 2：FSM 状态诊断（认知环定位）

### 定性模式

根据用户对当前状态的描述，判定 FSM 三态中的哪一个：

| 信号 | S₀ 大幅修正 | S₁ 在线微调 | S₂ 稳态 |
|------|:--:|:--:|:--:|
| 坐标系是否锚定 | ❌ 未锚定 | ✅ 已锚定 | ✅ 已锚定 |
| 残差幅度 | >20% | 1.67%~20% | <1.67% |
| 主要矛盾是否识别 | ❌ | ✅ | ✅ |
| 自适应速率 k | 0.7~1.0（探索为主） | 0.3~0.7（混合） | 0~0.3（利用为主） |

诊断步骤：
1. 确认系统层次是否已选定
2. 确认主要矛盾是否已识别
3. 评估预期与实际之间的偏差量
4. 根据以上三项判定 FSM 状态

### 双通道门控检查

当用户声称"已经收敛"或"问题已解决"时，执行假稳态检测：

- **观测器通道**：残差是否真的收敛？趋势是否下降？
- **元认知通道**：坐标系是否已锚定？层次是否一致？

如果观测器通道通过但元认知通道未通过 → **假稳态风险**。这意味着"在错误层次上做了精确优化"。

### 定量模式

```bash
cd ~/Desktop/qiushi-formalization/fsm-prototype
python3 main.py --scenario false-steady
```

### 输出格式

```
📍 FSM 状态诊断:
  当前态: [S₀/S₁/S₂]
  坐标系: [已锚定(层次X)/未锚定]
  残差:   [轻微 <5% / 中等 5-20% / 严重 >20%]
  速率区间: [探索/混合/利用]

🚪 双通道门控:
  观测器通道: [✅/❌] — [残差状态]
  元认知通道: [✅/❌] — [坐标系状态]
  综合判定:   [可通过 / 假稳态风险 / 模型偏差 / 全面不收敛]

⏱️ 超时风险:
  已在当前态连续 [N] 周期
  最大允许: [T_max] 周期
  建议: [继续微调 / 回退到 S₀ 重选坐标系 / 检查停滞原因]
```

## 工具 4：三级诊断框架（残差分类）

当用户提供残差观测值时，按以下框架分类并路由：

```
残差特征                    诊断          响应路径
─────────────────────────────────────────────────────
幅度 < γ₁ ∧ 方向散(dc<0.6)   Level 1 噪声   门控统计过滤，不触发修正
幅度 ∈ [γ₁,γ₂] ∧ 方向一致    Level 2 漂移   降阶校正——只估计偏差项
幅度 > γ₂ ∧ 方向一致          Level 3 结构   全重跑——认知模型重建
```

其中 γ₁=5%，γ₂=20%，dc 为方向一致性指标。

## 不适用场景

- 简单单步任务（不需要形式化分析）
- 纯事实查询（没有系统结构需要分析）
- 子 agent 执行的具体操作（子 agent 跳过此 skill）
- 用户没有提供任何系统描述或数据

## 常见错误

| 错误 | 纠正 |
|------|------|
| 在没有子系统描述的情况下强行构造耦合矩阵 | 先调用 investigation-first 摸清子系统边界 |
| 忘记标注数据缺口就用定量模式输出数值 | 缺失数据必须标注，给出"如果 X 则 Y"条件推演 |
| 把定性模式的推演结果当成精确计算 | 定性模式的所有数值都是示意——必须在结论中标注"定性推演，未经数值验证" |
