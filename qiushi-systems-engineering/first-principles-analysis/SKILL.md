# 第一性原理分析

> "每个系统的运动状态的特性与描述这个系统的微分方程的类型是有密切关系的。不但如此，微分方程的类型还确定了解决系统的工程问题的正确作法。"
> —— 钱学森、宋健《工程控制论》第1章

## 核心原则

**不类比、不套框架、不从"别人怎么做"出发。把问题拆解到不可再分的基本元素，然后从这些元素重新向上推理，构建理解或方案。**

> 详细原著依据见《工程控制论》第1章——钱学森不按"应用领域"（航空、化工、电气）分类控制系统，而是按描述系统的**微分方程基本类型**（线性/非线性、常系数/变系数、集中参数/分布参数）分类。方程类型是"第一性"——它决定了系统的全部行为特性和可用的分析方法。这种"从数学本质而非工程表象出发"的分类法，正是第一性原理思维在工程控制论中的集中体现。

## 不适用场景

以下情况**不需要**调用本 skill：
- 问题已有成熟框架且验证有效——直接用，不要为了"本质"去推翻
- 时间紧迫且类比法可以快速给出足够好的解
- 当前处于执行层，不需要重新理解问题本质
- 问题的基本信息尚未收集——先用 `investigation-first` 摸清现状

## 何时使用

- 面对一个真正新的问题，不知道该用什么方法
- 当前方案都是"参照XX做的"，想找有没有更根本的解法
- 假设已经堆积了多层（"因为A所以B，因为B所以C..."），不清楚底层还成立吗
- 需要区分"这个约束是物理上必须的"还是"行业惯例"
- 多个专家给了互相矛盾的建议，需要回到原点重新判断
- 想从源头推导出方案，而非在别人的方案上修补

## 与矛盾分析法的分工

`contradiction-analysis` 和本 skill 是 Layer 1 的双子星：

| | 第一性原理分析 | 矛盾分析法 |
|---|---|---|
| 追问 | "最基本的事实是什么？" | "核心的张力在哪里？" |
| 切入 | 向下挖到不可再分的根基 | 横向着眼于对立面的统一和斗争 |
| 产出 | 问题的基本元素和本质约束 | 主要矛盾及其主要方面 |
| 时序 | **先**——先理解本质 | **后**——在本质基础上找矛盾 |

> 典型流程：`first-principles-analysis`（"这个系统本质上是什么"）→ `contradiction-analysis`（"它的主要矛盾是什么"）→ `investigation-first`（"去调查验证"）。

## 方法流程

### 第一步：剥离（Strip）

列出当前关于该问题的所有已知陈述。逐条打标签：

| 标签 | 含义 |
|------|------|
| **物理约束** | 自然规律决定的，不可违反 |
| **逻辑必然** | 由物理约束+定义推导出的必然结论 |
| **场景约束** | 由系统的使用场景施加的硬边界。谁用、在哪用、怎么用、多久用一次——这些答案直接决定了系统必须满足什么条件。对产品系统而言，这跟物理约束一样不可违反 |
| **经济约束** | 由系统的存续条件（市场、预算、成本结构）施加的边界。产品必须能在其经济生态中存续，否则物理上再完美也没有意义 |
| **工程惯例** | 行业内约定俗成但非必然 |
| **历史遗留** | 过去某个条件下的选择，条件已变 |
| **类比借用** | "因为XX领域这么做，所以我们也这么做" |
| **未验证假设** | 可能成立但未经严格检验的前提 |

**输出**：如果讨论对象是物理系统（无用户、无使用场景），只保留前两类（物理约束 + 逻辑必然）。如果讨论对象是产品系统，还需保留场景约束和经济约束——它们对产品而言同样是不可违反的第一性约束。其余标签全部暂时搁置为"可质疑项"。

### 第二步：分解（Decompose）

将问题分解到**不可再分的基本元素**：

- 这个问题由哪些最基本的实体构成？
- 它们之间最基本的相互作用是什么？
- 描述这些实体和相互作用需要什么最基础的数学/逻辑形式？

**判断"到底了"的标准**：继续分解下去不再增加对问题的理解，或者分解后的元素已经可以用一句不含"因为"的话描述。

> 钱学森《工程控制论》的做法：任何控制系统 → 四个基本元素（受控对象 + 执行机构 + 放大器 + 敏感元件），每个元素用一个微分方程描述。这就是分解到"第一性"。

### 第三步：重建（Rebuild）

从基本元素出发，只使用物理约束和逻辑必然，向上推导：

1. 这些基本元素能组合出什么结构？
2. 在给定的物理约束下，哪些结构是可能的？哪些是不可能的？
3. 从可能的解中，选出对当前目标最优的

**纪律**：重建过程中不得引用第1步被搁置的工程惯例、历史遗留和类比借用——除非重建结果和它们恰好一致。

### 第四步：回验（Cross-check）

将重建的结果与搁置的惯例/类比对碰：

- **一致**：惯例有第一性依据，增强了信心
- **不一致**：可能是惯例的隐藏前提已失效 → 有创新机会
- **证明惯例不可能**：第一性推导给出了上限 → 惯例可能是错的或条件不同

## 与其他 skill 的关系

- **前置**：`first-principles-analysis` → `contradiction-analysis`（理解本质后再找矛盾）
- **并行**：与 `investigation-first` 双轨——调查获取事实，第一性分析穿透事实到本质
- **下游**：重建结果送入 Layer 2（`systems-thinking-framework` → 建模）或 Layer 3（`hierarchical-decomposition-coordination` 等求解技能）
- **循环**：重建的方案通过 `practice-cognition` 回到实践检验，如果实践与推导不一致，重新审视第一步中被判定为"物理约束"的东西是否真的是物理约束

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 把"行业最佳实践"当做物理约束 | 物理约束是自然规律，行业实践是社会建构，二者严格区分 |
| 分解不够深就停止 | 每个基本元素应该能用"不含因为的句子"描述 |
| 重建时偷渡了之前搁置的假设 | 重建只用第一步保留的物理约束+逻辑必然 |
| 认为第一性原理就是"不学别人" | 第一性原理不是拒绝学习——是学完之后能独立推导，不盲从 |
| 对于已经有成熟理论的问题也从头推 | 成熟理论正是前人做过第一性分析的结果，直接用 |
