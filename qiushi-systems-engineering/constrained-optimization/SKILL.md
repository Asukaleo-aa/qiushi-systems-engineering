# 约束优化——最优决策理论

> "当控制量受限制时，线性系统内的最速控制必定是开关式函数……最优控制 $\mathring{\boldsymbol{u}}(t)$ 在任何时刻都使 $(\boldsymbol{\psi}(t), B\boldsymbol{u}(t))$ 获得极大值。"
> —— 钱学森、宋健《工程控制论》第8章

> "无论系统的初始状态如何，也无论第一阶段运动过程中的最优控制是何种形式，全局为最优的控制函数，对应于第一阶段的终点状态，第二阶段的选择也一定是最优的。"
> —— 钱学森、宋健《工程控制论》第9章

## 核心原则

**任何决策问题，当目标明确、行动空间有边界、每一步选择都会改变可行空间时，存在数学上可定义的最优策略。找出这个策略，不是靠枚举也不是靠直觉，而是靠结构化的最优性条件。**

在钱学森和宋健的框架中，最优控制从古典变分法发展到庞特里亚金极大值原理和贝尔曼动态规划，背后有一条贯穿的方法论线索：**最优决策具有可继承的子结构——整体最优意味着每一步的最优，并且最优解通常触碰约束边界而非停留在内部。**

---

## 不适用场景

以下情况**不需要**调用本 skill：
- 目标本身模糊，还没有定义清楚"好"的标准（先用 `first-principles-analysis` 澄清目标）
- 选项之间不存在可比较的共同度量
- 约束是软性的、谈判性的而非硬性的物理/逻辑限制
- 决策时间窗口极短，来不及形式化
- 不确定性的影响远大于优化空间的影响（先用 `investigation-first` 摸清底细）

## 方法论流程

### 第一步：清晰定义目标与约束

从钱学森的分类出发，先明确问题的类型：

**目标函数层次校验（在定义目标之前必做）：**
- 这个系统是物理系统、工程系统、还是产品系统？（见 `systems-thinking-framework` 第零步）
- 如果是产品系统，当前定义的目标是"终端目标"（用户关心的最终度量）还是"中间指标"（物理/技术层面的局部最优）？
  - 例：Isp 最大化在物理层是终端目标，在产品层是中间指标。产品层的终端目标是 $/kg to orbit 或任务成功率
- 如果目标定义在错误层次上，最优解可能物理上正确但在产品语境下无意义

**目标函数**（要最大化或最小化的东西）：
- 能否写成单一可比较量？（如时间、成本、收益、误差）
- 是积分型（过程累加）还是终端型（只看最终结果）？

**约束空间**：
- 控制变量（你的行动自由度）是什么？
- 每个控制变量的上下界是什么？——A 型限制（分量独立受限）还是 B 型限制（总量受限）？
- 哪些约束是物理/逻辑上不可逾越的？哪些是惯例？

**输出格式：**
```
目标：[最小化/最大化] [量]
控制变量：[列表，每个的可行区间]
硬约束：[不可违反的限制]
系统状态：[受决策影响的变量及其演化规则]
```

### 第二步：判断问题的结构类型

钱学森将最优控制问题按系统类型分类——这个分类决定了解题路径：

| 类型 | 特征 | 适用的解思路 |
|------|------|------------|
| 线性系统 + 限制型约束 | 受控对象线性，控制受上下界约束 | 极大值原理 → 开关型（Bang-Bang）控制 |
| 线性系统 + 二次型指标 | 指标为平方误差积分，无控制约束 | 黎卡提方程 → 线性反馈 |
| 非线性 + 一般约束 | 系统或指标为非线性 | 标准方程组 + 两点边值问题 |
| 离散阶段决策 | 决策在离散时间点做出 | 动态规划 |
| 模型不确定 | 系统参数未知 | 跳到 `adaptive-robust-strategy` |

**关键判断**：如果控制变量受限（如预算有上限、时间有上限），最优解通常不是"中等力度"，而是**极端值**——要么满额使用，要么零。这就是极大值原理的方法论内涵：最优控制触碰约束边界。

### 第三步：选择解法并求解

#### 路径 A：极大值原理（适合连续控制问题）

核心思想来自 8.2 节：最优控制 $\mathring{\boldsymbol{u}}(t)$ 在每一瞬间都最大化哈密顿函数
$$
H = (\boldsymbol{\psi}, A\boldsymbol{x} + B\boldsymbol{u})
$$

对 agent 来说，操作化为：
1. 写出你关心的"状态变量"的演化方程（即使不精确也要写出形式）
2. 引入"伴随变量"——它衡量状态变化对最终目标的边际影响（影子价格）
3. 在每个瞬间，选择行动使"控制对目标的边际贡献"最大化

**实际应用**：当你需要在多个方案间分配有限资源时，把每个方案的"边际产出"算出来，按边际产出从高到低分配资源，直到预算耗尽。这就是极大值原理的离散版。

#### 路径 B：动态规划（适合阶段性决策问题）

核心思想来自 9.10 节：最优路径的任何一个后缀子路径，对于该子路径的起点而言也是最优的。

操作步骤：
1. 从最后一阶段倒推：在最后一阶段，对每个可能的进入状态，找到使末端指标最优的决策
2. 逐阶段向前递推：每个阶段的决策 = 当前阶段的立即收益 + 后续阶段已经算出的最优继续值
3. 到初始状态时，全局最优路径自然浮现

**实际应用**：当你面对"分几步推进一个项目，每一步的选择影响下一步的可行空间"时——不要正向枚举所有可能性，而是从目标倒排。

### 第四步：检验最优性

- 满足 "极大值原理" 检验：每个时刻/阶段的决策是否确实在所给约束下取得了极值？
- 满足 "动态规划" 检验：把最优路径从中间截断，后半段对于那个中间点是否仍然最优？
- 边界检验：最优解是否触碰了某个约束？如果所有约束都松弛，检查目标函数形式是否合理。

### 第五步：转化为可执行方案

将数学上的最优解映射回实际动作：
- 哪个约束是真正的瓶颈？对它做灵敏度分析
- 最优解距离次优解有多远？如果很近，选哪个都可以；如果很远，必须盯住最优
- 执行中需要监控哪些指标来确保不偏离最优轨线？

```
## 最优决策分析报告

### 1. 问题公式化
[目标函数 / 控制变量 / 约束条件 / 状态方程]

### 2. 问题类型诊断
[线性/非线性, A型/B型约束, 连续/离散]

### 3. 选用的解法及理由
[极大值原理 / 动态规划 / 组合优化 / ...]

### 4. 最优策略
[最优决策的表述——不是数学推导，是可执行的行动规则]

### 5. 瓶颈约束
[哪个约束限制了进一步优化？它的影子价格是多少？]

### 6. 灵敏度
[如果约束放松 10%，目标改善多少？哪些约束值得争取放宽？]

### 7. 次优方案
[如果最优不可行，退一步的次优方案是什么？代价多大？]
```

## 黄金法则

1. **触碰边界**：资源受限时，最优解通常把限制性资源用到极限——而非均匀分配
2. **倒推优先**：阶段性决策问题，从目标往回推比从起点往前枚举高效得多
3. **边际相等不成立时有角点解**：最优不一定在"边际收益相等"处，而可能在约束边界
4. **影子价格指导取舍**：每个约束都有影子价格——它告诉你"多一单位这个资源值多少"，据此决定争取谁
5. **不追求精确模型才用这招**：如果问题足够简单可以直接算，直接算；这个 skill 的价值在于结构复杂、直觉不够用时提供框架

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 在不清楚目标时就开始优化 | 先定义可度量的目标函数，没有它就无从谈"最优" |
| 对受限变量取中间值 | 受限变量在最优解中通常触碰边界（"Bang-Bang"），除非有严格的凸理由让它取内点 |
| 正向枚举所有可能 | 阶段性决策用动态规划倒推；枚举是指数爆炸 |
| 忽略约束之间的耦合 | B 型限制（总量受限）和 A 型限制（分量独立受限）最优解完全不同 |
| 只做一次优化就不管了 | 条件变化了要重新解；最优是相对于给定约束和目标的 |

## 与其他 skill 的关系

**认知位置**：求解环。在「建模」环产出系统模型后启用。

**上游输入**：
- `first-principles-analysis`：明确问题的本质约束（区分物理约束和惯例约束）
- `contradiction-analysis`：主要矛盾 = 瓶颈约束 = 优先放松的约束
- `state-estimation`：估计当前系统状态，为优化提供初始条件

**同环工具**：
- 正向推导碰壁 → `duality-complementarity-analysis`（构造对偶问题求解）
- 直接求解不可行 → `perturbation-progressive-method`（渐进逼近）
- 工具不趁手 → `multi-representation-equivalence-transform`（切换数学表述）
- 多子系统优化 → `hierarchical-decomposition-coordination`（分解协调）

**下游交付**：
- 最优策略输出 → `engineering-orchestration` 编排为可执行任务
- 最优方案 → `practice-cognition` 循环在实践中检验
