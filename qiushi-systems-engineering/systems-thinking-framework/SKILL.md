# 系统思维框架

> "系统概念来源于古代人类的社会实践经验，所以一点也不神秘。人类自有生产活动以来，无不在同自然系统打交道。"
> —— 钱学森《系统思想和系统工程》

## 核心原则

**系统不是部分的简单加和。凡是复杂任务，必须先建立系统世界观，再动手。不许在没有系统思维的情况下直接进入执行。**

在钱学森的理论体系中，系统思维是整个系统工程的起点。他从古代都江堰、《黄帝内经》、齐国名医扁鹊的综合辨症等历史案例中提炼出：人类在知道系统思想之前，就已经在进行辩证的系统思维。本 skill 将这些原则转化为 AI agent 的操作规程。

---

## 不适用场景

以下情况**不需要**调用本 skill：
- 任务范围极小（修改一行代码、回答一个简单问题）
- 任务已经在已有的系统框架内，且当前步骤是纯执行
- 用户明确要求"直接做，不要想太多"
- 紧急修复任务且问题边界完全明确

## 方法论流程

### 第零步：确定系统层次

> "系统工程就是从系统的认识出发……" —— 但在认识系统之前，先要认清：你是在哪个层次上讨论这个系统？

同一个对象在不同的系统层次上，目的不同、边界不同、最优解不同。跳过了这一步，所有的后续分析都可能在错误层次上运行。

**系统层次分类：**

| 层次 | 系统定义 | 目标的来源 | 典型约束 |
|------|---------|----------|---------|
| **物理系统** | 由自然定律封闭的对象 | 内部物理量（效率、比冲、功率密度） | 守恒定律、材料本征极限 |
| **工程系统** | 物理系统 + 制造/装配/维护可行性 | 性能-可制造性权衡 | 工艺极限、供应链、公差 |
| **产品系统** | 工程系统 + 使用场景 + 成本结构 + 生命周期 | **使用场景赋予的外部目的**（单位成本、任务成功率、翻修周期） | 用户需求、经济可行性、竞争生态 |
| **社会-技术系统** | 产品系统 + 组织/市场/制度环境 | 长期存续与演化能力 | 产业生态、法规、资本效率 |

**判断规则：**
- 如果系统有明确的终端用户，它就是产品系统——目标不由内部指标定义，而由用户需要定义
- 如果系统是某个更大系统的子组件，它的目标必须由母系统的目标向下分解得出
- 仅在系统完全自封闭（无用户、无母系统）时，才可以用内部物理/技术指标作为目标

**输出格式：**
```
系统层次：[物理系统 / 工程系统 / 产品系统 / 社会-技术系统]
母系统（如果有）：[这个系统服务于哪个更大的系统？]
层次选择依据：[为什么选这个层次？] 
```

### 第一步：明确系统目的

> "系统工程就是从系统的认识出发，设计和实施一个整体，以求达到我们所希望的效果。" —— 钱学森《系统思想和系统工程》

先回答：**这个系统要达成什么目的？**
- 目标是什么？（用一句话定义系统使命——必须与第零步选定的系统层次一致）
- 成功的标准是什么？（可量化的指标——来自第零步确定的"目标的来源"）
- 对谁而言是成功？（利益相关方视角——由系统层次决定谁是利益相关方）

**输出格式：**
```
系统使命：[一句话]
成功指标：[可量化标准列表]
关键约束：[不可违反的硬约束]
```

### 第二步：界定系统边界

> "什么叫系统？系统就是由许多部分所组成的整体，所以系统的概念就是要强调整体，强调整体是由相互关联、相互制约的各个部分所组成的。"

- 什么在系统之内？（可控、可直接影响的组分）
- 什么在系统之外？（环境、不可控的外部条件）
- 系统与环境之间有什么交换？（输入、输出、约束）

**输出格式：**
```
系统内：[组分列表]
系统外（环境）：[环境因素列表]
交换接口：[输入列表] | [输出列表]
```

### 第三步：识别组分及其关联

> "系统概念来源于古代人类的社会实践经验...古代农事、工程、医药、天文知识等方面的成就，都在不同程度上反映了朴素的系统概念的自发应用。"

钱学森强调系统思维的关键在于看到**关联**，而非孤立的部分。
- 系统由哪些组分（子系统）构成？
- 组分之间如何相互作用？（依赖、制约、增强、反馈）
- 哪些关联是关键路径？哪些是次要的？

**输出格式：**
```
组分A → [作用类型] → 组分B →
组分B → [作用类型] → 组分C →

关联矩阵：
        A    B    C    D
     A  -   +1    0   -1
     B  +1   -   +2    0
     C   0   -2    -   +1
     D  +1    0   -1    -
（+表示增强/依赖，-表示制约/冲突，0表示无直接关联，数值表示强度）
```

### 第四步：分析层次结构

> 钱学森提出"现代科学技术的四层次结构"：工程技术 → 技术科学 → 基础科学 → 马克思主义哲学

将你的系统按类似方式分层：
- **执行层**：具体做什么、怎么做
- **方法层**：用什么方法、什么工具
- **理论层**：依据什么原理、什么规律
- **哲学层**：指导思想、根本假设

**输出格式：**
```
哲学层（为什么）: [根本假设、指导原则]
理论层（是什么）: [核心原理、基本规律]
方法层（用什么）: [技术方法、分析工具]
执行层（怎么做）: [具体步骤、操作流程]
```

### 第五步：发现涌现特性

> "量变可以引起质变：H.Haken等人的协同学证明这是可能的，即巨系统的统计理论说明巨系统中会出现简单系统中没有的现象，如自组织现象。" —— 钱学森《关于系统学的通信》

- 单独看组分不具备、但组合后系统具有的特性是什么？
- 哪些涌现特性是预期的？哪些可能是意外的？
- 什么条件下可能出现涌现？

**输出格式：**
```
预期涌现特性：[列表]
潜在意外涌现：[列表]
触发条件：[对每个涌现特性的条件说明]
```

### 第六步：建立反馈结构

> "控制论是研究系统控制及其应用的科学。也就是研究如何分析、综合和组成系统，研究系统各个组成部分之间的关联和制约关系。" —— 于景元《控制论和系统学》

- 系统中有哪些反馈回路？
- 哪些是正反馈（放大）？哪些是负反馈（稳定）？
- 反馈的延迟时间是多少？

**输出格式：**
```
正反馈回路（放大）:
  A↑ → B↑ → A↑ （增强型）
负反馈回路（稳定）:
  C↑ → D↓ → C↓ （调节型）
关键延迟：[哪些反馈有显著滞后]
```

## 强制输出

在完成以上六个步骤后，必须输出一份**系统思维报告**：

```
## 系统思维报告：<系统名称>

### 1. 系统目的
[系统使命、成功指标、关键约束]

### 2. 系统边界
[系统内组分 / 环境因素 / 交换接口]

### 3. 组分关联
[关联矩阵或关联图 + 文字说明]

### 4. 层次结构
[四层分解]

### 5. 涌现特性
[预期 + 潜在意外]

### 6. 反馈结构
[正反馈 / 负反馈 / 延迟]

### 7. 关键未知项
[当前不确定、需要后续调查研究的内容]

### 8. 后续行动建议
[基于系统思维，应该先做什么后做什么，应调用哪些后续 skill]
```

## 黄金法则

1. **整体先于部分** —— 先看到森林，再研究树木
2. **关联重于孤立** —— 组分间的线比组分本身更重要
3. **目的决定结构** —— 系统的结构服务于系统的目的
4. **层次不可跳过** —— 每个层次都有不可替代的规律
5. **涌现无法还原** —— 整体特性不能仅从部分特性推导

## 与其他 skill 的关系

### 系统工程链内调用

本 skill 是**全部系统工程 skill 链的工程入口**。完成后通常按以下顺序调用：
1. → `system-boundary-structuring`（如果需要更精细的结构分析）
2. → `problem-domain-investigation`（如果需要深入调研）
3. → `meta-synthesis-engine`（当需要整合多个信息来源时）
4. → `quantitative-modeling-workflow`（当系统理解已足够进入建模阶段）

### 与 qiushi 思想方法层的整合

本 skill 的哲学层（第四步）建立在 qiushi 方法论之上。在调用本 skill 之前或同时，应优先由 `arming-thought` 的路由判断触发以下 qiushi skill：

| 本 skill 的步骤 | 联动 qiushi skill | 联动方式 |
|---------------|------------------|---------|
| 第一步（系统目的） | `overall-planning` | 多目标动态平衡→系统使命的权衡 |
| 第三步（组分关联） | `contradiction-analysis` | 关联矩阵中的正负信号→矛盾图谱 |
| 第四步（层次结构） | `arming-thought` | 哲学层的"实事求是"是最高指导原则 |
| 第六步（反馈结构） | `practice-cognition` | 反馈回路→实践认识螺旋的实现机制 |
| 第七步（关键未知项） | `investigation-first` | 未知项→调研提纲的生成 |

### 统一工作流入口说明

本 skill 是系统工程**技术侧的入口**。从对话全局看，完整的入口链是：

```
arming-thought （实事求是总判断 + 路由）
  ├→ contradiction-analysis （如果主要矛盾不清晰）
  └→ systems-thinking-framework （本 skill，如果系统复杂需要整体把握）
        ├→ system-boundary-structuring
        ├→ problem-domain-investigation
        ├→ meta-synthesis-engine
        └→ ...
```

**禁止**：在未完成本 skill 的情况下，直接进入代码编写或具体执行。
