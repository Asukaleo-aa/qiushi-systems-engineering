# 对偶互补分析

> "由证明可知……系统的能观测性等价于其对偶系统的能控性。"
> —— 钱学森、宋健《工程控制论》第4章

## 核心原则

**每一个正向问题都存在一个对偶的反向问题。正向路径被卡住时，切换到对偶路径往往获得新的数学结构或直观洞察。**

对偶不是"相同的另一个"，而是"互补的另一面"：
- 状态 ↔ 协态（运动 ↔ 影响）
- 控制 ↔ 观测（施加影响 ↔ 获取信息）
- 向前时间 ↔ 向后伴随（因果 ↔ 追溯）
- 代价 ↔ 收益（支出 ↔ 所得）
- 源 ↔ 汇（产生 ↔ 吸收）

> 详细原著依据见《工程控制论》第4.10—4.11节（能控能观测对偶）、第8章（极值原理中的伴随方程对偶）、第13章（摄动理论中的伴随函数对偶）。

## 不适用场景

以下情况**不需要**调用本 skill：
- 问题只有一个明确的单向因果方向
- 不存在任何形式的对称/互换结构
- 正向方法已经高效解决，无需绕路
- 对偶变换的成本超过收益（简单问题）

## 何时使用

- 正向求解某问题时遇到不可逾越的障碍
- 问题有天然的对称结构：输入/输出、原因/结果、状态/观测
- 需要从可观测数据推断不可测状态（状态估计/滤波问题）
- 需要在"控制成本"和"观测精度"之间做互换
- 优化问题中原始变量很多但对偶变量很少

## 方法流程

### 第一步：识别原问题的数学结构

将当前问题写成标准形式。以最常见的几种为例：

| 原问题类型 | 标准形式 |
|-----------|---------|
| 控制系统 | ẋ=Ax+Bu, y=Hx |
| 优化问题 | min f(x) s.t. g(x)≤0 |
| 观测量与状态 | y=Hx+噪声, 求 x̂ |
| 因果链 | A→B→C→D |

### 第二步：构造对偶系统

按原问题类型选择对应的对偶变换：

| 原问题 | 对偶问题 | 变换方式 |
|--------|---------|---------|
| 能控性 (A,B) | 能观测性 (A^T,B^T) | 矩阵转置 + 角色互换 |
| 状态方程 ẋ=Ax+Bu | 伴随方程 ψ̇=-A^Tψ | 时间反向 + 转置 |
| 原优化 (min) | 拉格朗日对偶 (max) | 引入乘子，构造鞍点 |
| 根轨迹零点 | 根轨迹极点 | 1/F(s) → F(s) |
| 向前传播路径 | 向后追溯路径 | 反转因果方向 |
| 信息发送 | 信息接收 | 香农信道输入↔输出对称 |

**关键判断**：对偶系统是否更简单？
- 状态维数是否从高维降为低维？
- 约束是否从非线性变为线性？
- 是否从无穷维变为有限维？

### 第三步：评估对偶间隙

确定对偶问题的最优解与原问题是否一致：

- **强对偶成立**（零间隙）：凸优化+Slater条件 / 线性规划 → 可直接用对偶解
- **弱对偶**（有间隙）：非凸 → 对偶解提供下界/上界，作为原问题的参考界限
- **无直接数值对偶**：定性互补 → 从对偶获得几何直观/结构理解，不一味追求数值解

### 第四步：解对偶问题

在对偶空间中求解。使用与原问题相同或更高效的数学工具。

### 第五步：翻译回原问题

将对偶解用原问题的语言重新表述：
- 对偶变量 λ* → 原约束的影子价格
- 协态 ψ(t) → 状态 x(t) 对指标的灵敏度
- 伴随系统输出 → 原系统输入的"影响力"

> **翻译不回去说明对偶构造有误**。返回第二步重新检查变换。

## 与其他skill的关系

**认知位置**：求解环。当正向求解受阻时启用。

**同环工具**：
- 对偶构造后需要选计算工具 → `multi-representation-equivalence-transform`
- 对偶解做渐进精化 → `perturbation-progressive-method`
- 对偶问题可能本身就是一个约束优化 → `constrained-optimization`

**回连上游**：
- 能控↔能观测对偶的直接应用 → `state-estimation`（状态估计 = 能观测性的工程实现）
- 对偶间隙的存在说明模型不确定性 → `adaptive-robust-strategy`
- 对偶解的验证遵循 `practice-cognition` 的实践检验原则

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 认为对偶就是"反过来做一遍" | 对偶变换有明确的数学规则，不是随意反转 |
| 不检查对偶间隙就用对偶解替代原解 | 非凸问题有间隙，只能做参考界限 |
| 解了对偶但翻译不回去 | 每一步变换都保持等价映射关系 |
