# 定量建模工作流

> "系统工程是当代人类在认识和改造客观世界中的一个'飞跃'。这种现代科学技术的定量方法，不但可以用于自然科学，而且可以用于经济事务的组织管理。"
> —— 钱学森《关于系统工程与经济管理体制》

> "提出经验性假设（猜想或判断），是建立复杂行为系统数学模型的出发点。"
> —— 钱学森《论系统工程》增订版说明

## 核心原则

**模型不是现实的完整复制，而是对关键关系的抽象。所有模型都是错的，但有些是有用的。从定性认识到定量模型，是系统工程最关键的"飞跃"——必须可追溯、可验证、可修正。**

---

## 不适用场景

以下情况**不需要**调用本 skill：
- 纯定性分析不涉及数值计算
- 问题过于简单（可以直接估算）
- 无法获得建模所需的基本数据或参数
- 系统的定性理解还不充分（应先调用 systems-thinking-framework）

## 方法论流程

### 第一阶段：从定性到定量——建立经验性假设

> 钱学森方法论的关键入口："提出经验性假设（猜想或判断），是建立复杂行为系统数学模型的出发点。"

**将系统思维报告中的定性理解转化为可量化的假设：**

```
对于系统的每个关键关系，写出定量假设：

定性描述 → 定量假设 → 可检验的预测

示例：
定性：温度升高会加速反应
定量：反应速率 k = A·exp(-Ea/RT)，其中Ea≈50±10 kJ/mol
可检验预测：当T从300K升至310K时，速率应增加约2倍

定性：系统规模增大效率降低
定量：效率 η = η₀ / (1 + αN)，其中α≈0.01±0.005
可检验预测：N=10时效率 0.91η₀，N=100时效率 0.50η₀
```

### 第二阶段：变量识别与参数估计

**完整列出模型涉及的所有变量：**

| 变量名 | 符号 | 单位 | 类型 | 来源/依据 |
|--------|------|------|------|----------|
| 变量1 | x₁ | m/s | 状态变量 | 从组分A的速度导出 |
| 变量2 | x₂ | N | 输入变量 | 外部施加 |
| 参数1 | p₁ | kg/m³ | 常数 | 材料手册，20°C时测量 |
| 参数2 | p₂ | 无量纲 | 待标定 | 需要通过数据拟合确定 |

**参数估计的三来源：**
1. **已知常数**：从权威来源直接获得（物理常数、标准值）
2. **经验公式**：基于已有研究的经验关系式
3. **数据拟合**：需要通过实验或历史数据标定

### 第三阶段：选择建模方法

**根据系统特性选择适当的建模方式：**

| 系统特性 | 推荐方法 | 书中的对应理论 |
|---------|---------|-------------|
| 连续、可微、确定 | 微分方程（ODE/PDE） | 工程控制论（第47章） |
| 离散事件、随机 | 离散事件仿真 | 军事系统工程（第3章） |
| 优化问题 | 运筹学（规划/对策/网络） | 系统工程（第1章、第12章） |
| 不确定性显著 | 概率模型/随机过程 | 控制论（第44章） |
| 复杂系统涌现 | Agent-based/系统动力学 | 系统学、协同学（第17章、第44章） |
| 定性与定量混合 | 灰色系统/模糊逻辑 | 大系统理论（第39章） |

### 第四阶段：模型构建

**按以下模板构建模型：**

```
模型名称：

域（Domain）：
[模型的适用范围和边界条件]

变量声明：
  输入：x ∈ R^n（可控/外部）
  状态：s ∈ R^m（系统内部状态）
  输出：y ∈ R^k（观测到的结果）
  参数：θ ∈ R^p（标定或估计）

核心关系：
  状态方程：ds/dt = f(s, x, t; θ)
  输出方程：y = g(s, x; θ)
  约束条件：h(s, x, y; θ) ≤ 0

  关键函数的具体形式：
  f(·) = [方程形式，解释每一项的物理/逻辑意义]
  g(·) = [方程形式]
  h(·) = [约束形式]

简化假设（必须显式列出）：
  1. [假设1]：简化了什么？为什么可接受？对精度的影响预估值
  2. [假设2]：...

参数值：
  θ₁ = [值] ± [不确定度]（来源：[依据]）
  θ₂ = ...

数值方法：
  积分方法：[如RK4、Euler]，步长：[Δt]
  优化方法：[如梯度下降]，收敛判据：[ε]
  随机采样：[如MCMC]，样本数：[N]
```

### 第五阶段：模型验证

> 钱学森强调系统工程"要强调实干，取得实际效果，不是光研究学问"

**多级验证方案：**

1. **内部一致性检查**：
   - 量纲分析（所有方程两边量纲一致）
   - 边界情况（极端参数值下行为是否合理）
   - 守恒律（质量/能量/信息是否守恒）

2. **敏感性分析**：
   - 哪些参数对结果影响最大？
   - 这决定了标定的优先级

3. **历史数据对比**（如有）：
   - 用已知数据检验模型预测能力
   - 误差分析：偏差 vs 方差


## 常见建模错误

| 错误 | 表现 | 书中的警示 |
|------|------|----------|
| 过度简化 | 忽略关键非线性效应 | "仅仅停留在分析的方法上，是不够的" |
| 虚假精度 | 给出6位小数但参数不确定度达50% | 计量系统工程：精度必须可追溯 |
| 黑箱崇拜 | 只用复杂模型而不理解其内部逻辑 | 控制论强调理解内部关联 |
| 忽略反馈 | 没有考虑系统内生反馈 | 控制论核心是反馈 |
| 参数无来源 | 使用"默认值"但不说明出处 | 标准化：所有数据须有依据 |

## 与其他 skill 的关系

### 系统工程链内

- **上游**：接收 `systems-thinking-framework` 的系统思维报告和 `system-boundary-structuring` 的结构分析
- **下游**：模型传递给 `simulation-validation-cycle` 进行仿真检验
- **反馈**：如果验证失败，需要返回本 skill 修正模型，或返回 `problem-domain-investigation` 补充数据

### 与 qiushi 思想方法层深度耦合

本 skill 的核心操作——"从定性到定量"的飞跃——正是 qiushi `practice-cognition` 中"感性认识→理性认识"飞跃的数学化和技术化实现：

| 本 skill 的阶段 | 对应的 practice-cognition 阶段 | 联动说明 |
|---------------|------------------------------|---------|
| 第一阶段（经验性假设） | 感性认识 → 理性认识 | "提出经验性假设"就是认识飞跃的起点 |
| 第二阶段（变量识别与参数估计） | 理性认识形成 | 将模糊的定性关系转化为精确的变量和参数 |
| 第三阶段（选择建模方法） | — | 纯技术步骤，由系统工程方法指导 |
| 第四阶段（模型构建） | 理性认识形成 | 将认识固化为可计算形式 |
| 第五阶段（模型验证） | 实践验证 | 回到实践检验——对应 `simulation-validation-cycle` |

**认识论链条：**
```
practice-cognition （宣告进入"理性认识形成"阶段）
    ↓
quantitative-modeling-workflow （定性→定量飞跃）
    ↓
simulation-validation-cycle （实践检验）
    ↓
practice-cognition （评估：成功→进入下一轮 / 失败→返回修正）
```

**其他 qiushi 联动：**
- `investigation-first`：参数来源不明时，触发实地调查
- `contradiction-analysis`：模型中的 trade-off（精度 vs 复杂度）→矛盾分析
- `overall-planning`：多目标优化时的权衡→统筹兼顾
