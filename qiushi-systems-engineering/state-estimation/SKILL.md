# 状态估计——系统辨识与信息融合

> "输入信号并不是确切知道的，只能用某些统计特性加以描述……机翼的应力是系统的输出，它也是一个随机函数，也只能知道它的统计特性。"
> —— 钱学森、宋健《工程控制论》第14章

> "卡尔曼提出了一种递推式滤波器，他不是从维纳-何甫方程出发，而是直接从信号模型出发，用递推的办法来求最优线性滤波器的结构和其中的参数。这种滤波方法的主要特点是适合用数字计算机进行运算，这对实时数据处理有很大的优点。"
> —— 钱学森、宋健《工程控制论》第15章

## 核心原则

**你看不到系统的全部，只能看到输出。从带噪声的输出中推断系统的内部状态——这是所有信息不完整决策的基础。钱学森的框架给出了两条路径：频域的维纳滤波（需要全貌统计）和时域的卡尔曼滤波（递推，每次只处理新数据）。它们的共同本质是：最优估计 = 信号在观测空间上的正交投影。**

---

## 不适用场景

以下情况**不需要**调用本 skill：
- 系统状态完全可直接观测且无噪声（不需要"估计"）
- 问题本质是逻辑推理而非统计推断
- 数据量极少，不足以支撑统计模型
- 信息可以一次性全部获取，不需要流式更新
- 不确定性来自对手的策略性行为而非随机噪声（那是博弈问题，不是估计问题）

## 方法论流程

### 第一步：区分信号与噪声

> 14.5 节：平稳随机函数的谱分解——任何平稳随机过程可以表示为不同频率正弦波的叠加，其功率谱密度揭示了能量在各频率上的分布。

操作化的核心问题：
- **什么是信号？** —— 你真正想知道的系统状态或参数
- **什么是噪声？** —— 干扰观测但不含有用信息的成分
- **噪声的性质是什么？** —— 是白噪声（各时刻独立）还是有色噪声（有时间相关性）？噪声的方差多大？
- **信号和噪声在什么域里最易分开？** —— 时域？频域？空间域？

**输出格式：**
```
信号定义：[你要估计的量]
观测定义：[你能观测到的量，含噪声]
噪声模型：[类型 / 方差 / 相关性]
信号与噪声的可分性：[在哪个域里信号和噪声的特征差异最大]
```

### 第二步：建立系统模型

卡尔曼滤波的关键思想（15.11 节）——不从原始观测直接反推信号，而是先建立信号自身的动力学模型：

- **状态方程**：信号（你要估计的东西）如何随时间演化？受什么驱动？
- **观测方程**：观测值如何由信号和噪声组合而成？

即使模型不精确（"不准确的模型也比没有模型好"），只要给出一个近似形式，递推滤波就能工作并逐渐修正。

**实际操作（对 agent）**：
```
状态变量：[你要追踪的量——进度、风险、效果...]
状态演化：[这个量在没有干预的情况下会怎么变？]
观测变量：[你能定期获取的指标]
观测模型：[指标如何反映真实状态？偏移因子+噪声]
```

### 第三步：选择估计策略

| 场景 | 推荐策略 | 来源 |
|------|---------|------|
| 所有数据已齐备，可离线分析 | 维纳滤波（频域），利用全部历史统计 | 第15.5节 |
| 数据流式到达，需要在线更新 | 卡尔曼滤波（时域递推），每次只用新数据 | 第15.11节 |
| 模型参数也未知 | 自适应滤波——同时估计状态和参数 | 第15.14节 |
| 信号是非平稳的（统计特性在变） | 有限记忆滤波器——只用最近 N 个观测 | 第15.7节 |
| 只需检测信号存在与否 | 最优检测滤波器——最大化信噪比 | 第15.9节 |

### 第四步：递推更新——卡尔曼滤波的实操化

对 agent 来说，卡尔曼滤波的操作框架不需要矩阵运算，核心是以下循环：

```
对每个新观测：
  ① 预测：基于当前状态估计，预测下一个状态和下一个观测
  ② 获得实际观测
  ③ 计算更新量 = 增益 × (实际观测 - 预测观测)
  ④ 更新状态估计
  ⑤ 更新估计的不确定性
```

其中，**增益**决定了新观测有多少权重：
- 观测噪声大 → 增益小 → 信旧估计更多
- 观测噪声小 → 增益大 → 信新观测更多
- 自身模型不确定 → 增益大 → 尽快吸收新信息

**agent 可直接使用的类比**：
- 你对接手一个复杂项目，初始理解不完整（状态不确定）
- 每周你看到新的输出指标（观测）
- 每个新指标都会修正你对项目状态的估计
- 修正幅度取决于：这个指标可靠吗？（噪声小 → 大幅修正）你已经很有把握了吗？（不确定性低 → 小修正）

### 第五步：估计不确定性量化

15.16 节：信号参数估计——不仅有"最佳估计值"，还有"这个估计有多可靠"。

**输出必须包含：**
- 点估计（最可能的真实值）
- 估计误差的方差或置信区间
- 如果估计错了方向，后果是什么？

**核心原则：宁可说"估计值是 X ± Y，置信度 Z%"，也不说"我认为就是 X"。**

## 强制输出

```
## 状态估计报告

### 1. 信号和噪声定义
[你要估计什么 / 你能观测什么 / 噪声来源和性质]

### 2. 系统模型
[状态方程（定性即可）/ 观测方程 / 模型中的不确定来源]

### 3. 选用的估计策略
[维纳/卡尔曼/有限记忆/自适应... 选择理由]

### 4. 当前最佳估计
[点估计 + 不确定性区间]

### 5. 信息缺口
[当前估计最需要什么新信息来减少不确定性？最优先获取哪个观测？]

### 6. 更新规则
[当有新信息到达时，如何修正估计？修正规则是什么？]
```

## 黄金法则

1. **模型先于数据**：没有模型的滤波是盲目的；先有状态方程和观测方程，哪怕是近似的
2. **递推优于批处理**：在信息流式到达的场景下，Kalman 递推比每次都重算要高效得多
3. **增益自适应**：当你对估计越来越有信心，对新观测的依赖降低；当你发现估计偏差在增大，提高增益
4. **正交投影本质**：最优估计在数学上就是信号在观测空间上的正交投影——误差与观测正交，意味着观测中的信息已被充分榨取
5. **不确定性和估计同样重要**：给出估计的同时要给出误差范围；只有点估计没有区间的结论在工程上没有决策价值

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 把观测直接当真实状态 | 区分"看到了什么"和"真实是什么"——观测 = 信号 + 噪声 |
| 所有信息源同等待遇 | 不同信息源有不同噪声方差——给低噪声源高权重 |
| 攒够数据再做分析 | 在线场景用递推滤波——每一条新数据都在当前最佳估计的基础上微调 |
| 认为"模型不准所以不用模型" | 不准的模型 + 滤波修正 > 没有模型直接猜 |
| 忽略信号的非平稳性 | 如果信号统计特性在变，用有限记忆或自适应方法 |

## 与其他 skill 的关系

**认知位置**：感知环。与 `investigation-first`、`problem-domain-investigation`、`mass-line` 同为「感知」工具，各自处理不同子任务。

**上游输入**：
- `investigation-first`：定义观测框架（看什么、怎么测）——先确定了观测变量，state-estimation 才能建立观测方程
- `problem-domain-investigation`：提供领域知识（噪声的可能来源、信号的典型特征）

**同环协同**：
- `investigation-first` 收集一手事实 + `state-estimation` 从数据推断状态 = 双轨感知
- `mass-line` 收集人的意见 + `state-estimation` 处理数据信号 = 人+数据双源

**下游交付**：
- 估计出的系统状态 → `constrained-optimization`（在估计状态上做最优决策）
- 估计的不确定性量级 → `adaptive-robust-strategy`（决定自适应速率）
- 估计质量的审视 → `criticism-self-criticism`（判断估计是否稳定可靠）
- 频域↔时域切换 → `multi-representation-equivalence-transform`
