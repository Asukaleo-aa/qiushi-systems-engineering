# 问题域调查研究

> "在任何一门现代精密科学中，对数量关系的研究是科学概念形成的基础，在社会科学中也不应例外。"
> —— 宋健《社会科学研究的定量方法》

> "没有调查，没有发言权。"
> —— 毛泽东《反对本本主义》

## 核心原则

**在没有充分调查研究之前，不允许做重大决策判断。这不是搜索一下了事，而是系统性地构建对问题域的知识图景——区分已知事实、合理推断和无知领域。**

本 skill 与 qiushi-skill 的 `investigation-first` 互为补充——后者侧重毛泽东调查研究的哲学方法论，本 skill 侧重系统工程中结构化的领域调研流程。

---

## 不适用场景

以下情况**不需要**调用本 skill：
- 领域知识已经充分（之前已完成系统调研并文档化）
- 问题范围极小且完全在已知领域内
- 纯技术实现任务，答案可以通过运行代码直接验证
- 用户已提供完整的上下文和知识基础

## 方法论流程

### 第一步：确定调研范围

> 钱学森在《用科学方法绘制国民经济现代化的蓝图》中强调，研究必须明确"对象和任务"。

**明确调研的边界和深度：**

```
调研目的：[为什么要做这个调研？要回答什么核心问题？]
调研范围：[涉及哪些领域/学科？]
调研深度：[需要到什么粒度？原理级/方案级/实现细节级？]
时间约束：[调研需要多少时间？]
信息类型：[需要理论/数据/案例/工具/对比/最佳实践？]
```

### 第二步：构建知识地图

**按系统思维框架（参考 systems-thinking-framework）组织调研：**

挖掘以下层次的信息：
- **哲学层**：该领域的基本假设、范式、世界观
- **理论层**：核心原理、基本规律、公认模型
- **方法层**：常用工具、标准流程、最佳实践
- **执行层**：实际案例、可操作方案、实现细节

### 第三步：多源信息采集

> 钱学森在《关于中国系统工程发展的若干侧面》中，回顾了系统工程在中国发展的多个支柱（运筹学、管理科学、控制论）和众多学会的活动，展示了多来源采集的方法。

**强制进行多来源交叉验证：**

| 信息源类型 | 示例 | 可靠性 | 获取方式 |
|-----------|------|--------|---------|
| 学术文献 | 论文、专著 | ★★★★ | 搜索、阅读 |
| 官方文档 | API文档、标准 | ★★★★ | 查阅 |
| 实践经验 | 案例、事故报告 | ★★★ | 收集 |
| 专家意见 | 论坛、访谈 | ★★-★★★★ | 引用 |
| 社区共识 | StackOverflow、GitHub Issues | ★★-★★★ | 搜索 |
| 推测/假设 | 理论推断 | ★-★★ | 推理 |

### 第四步：信息分类与标记

**对每一条关键信息进行标记：**

```
标记体系：
[事实]    - 有多个独立来源证实，可视为确定
[推断]    - 基于已知事实的合理推论，需要验证
[假设]    - 当前接受但未经严格检验的前提
[存疑]    - 来源单一或不可靠，需要进一步确认
[未知]    - 应该知道但目前完全不了解
[分歧]    - 不同来源给出不同结论，原因待查
```

### 第五步：知识缺口分析

**识别调研后仍存在的关键缺口：**

```
知识缺口清单：
- [ ] 缺口1：[描述 + 为什么重要 + 影响什么决策]
- [ ] 缺口2：[描述 + 为什么重要 + 影响什么决策]

对每个缺口评估：
- 是否可以接受这个缺口？(Yes/No)
- 如果Yes：需要什么假设来填补？
- 如果No：需要什么行动来获取知识？（实验？更多调研？）
```

```
## 领域调研报告

### 1. 调研目的与范围

### 2. 知识地图
[按四层结构组织的领域知识]

### 3. 关键发现
[最重要的5-10条发现，每条标注可靠性标记]

### 4. 信息来源清单
[表格：来源 | 类型 | 可靠性 | 涵盖范围 | 引用链接]

### 5. 知识缺口清单
[表格：缺口 | 重要性 | 接受/不接受 | 填补计划]

### 6. 矛盾与分歧
[不同来源的矛盾结论 + 可能原因分析]

### 7. 调研结论
- 已知/确定：[列表]
- 合理推断：[列表]
- 关键假设：[列表]
- 尚不知：[列表]

### 8. 对后续工作的影响
[基于调研结果，对系统设计或决策的具体影响]
```

## 调研质量检查表

- [ ] 每个关键发现都有至少2个独立来源确认？
- [ ] 区分了事实与观点？
- [ ] 识别了所有知识缺口并评估了可接受性？
- [ ] 检查了信息来源的时效性？
- [ ] 考虑了领域的多面性（不同学派、不同方法）？
- [ ] 调研范围没有过大（信息过载）也没有过小（信息不足）？

## 与其他 skill 的关系

### 系统工程链内

- **上游**：在 `systems-thinking-framework` 建立系统世界观后调用
- **平行**：与 `system-boundary-structuring` 平行进行
- **下游**：调研结论传递给 `meta-synthesis-engine` 综合集成

### 与 qiushi 思想方法层深度耦合

本 skill 与 qiushi 的 `investigation-first` 构成**双轨调查**——两者不是替代关系，而是互补关系：

| 阶段 | `investigation-first`（哲学轨） | `problem-domain-investigation`（工程轨，本 skill） |
|------|-------------------------------|--------------------------------------------------|
| 启动 | 声明调查目的（"我要弄清楚……"） | 确定调研范围（覆盖率边界） |
| 执行 | 列调查提纲 → 深入一线 → 亲手记录 | 构建知识地图 → 多源采集 → 信息可靠性标记 |
| 收束 | 实事求是结论："现状是……关键约束是……" | 结构化报告："已知/推断/假设/存疑/未知/分歧" |
| 衔接 | 结论进入 `practice-cognition` 实践验证 | 结论进入 `meta-synthesis-engine` 综合集成 |

**联合调用协议：**
1. `arming-thought` 路由判断 → 同时或先后触发两个 skill
2. `investigation-first` 保证哲学方向正确（不脱离实际、不预设结论）
3. 本 skill 保证覆盖完整（四层知识地图、多源交叉验证）
4. 两方结论对照：一致项确认为 `[事实]`；不一致项标注为 `[分歧]`，触发 `contradiction-analysis` 或 `meta-synthesis-engine`
5. 联合调查完成后，知识缺口清单合二为一，传递给下游
