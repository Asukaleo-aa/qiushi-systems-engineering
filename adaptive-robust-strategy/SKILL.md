# 自适应系统方法——在不确定中寻找并保持最优

> "进行控制设计时就不需要有关控制系统性质的确切知识，在这里我们采用在控制过程中不断测量的办法来代替预先了解控制性质的要求，这种系统就称为自动寻求最优运转点的控制系统。"
> —— 钱学森、宋健《工程控制论》第16章

> "系统本身就能够自动地改正设计中不能预料到的失谐或偶然发生的差错。形象地讲，似乎控制系统本身就能'理解'如何正确地行动。这很像一切生物的适应性机能。"
> —— 钱学森、宋健《工程控制论》第18章

## 核心原则

**在信息不完备时，放弃一次性完美设计。将方案设计为一个能够在运行中感知、试探、评价、修正的动态过程。钱学森的自适应方法论给出了三条路径：自寻优（不知道最优在哪，通过试探找到它）、自镇定（不知道哪个模式稳定，通过切换找到稳定态）、模型参考（不知道怎么做最好，但知道"对"的样子，向着它调整）。**

---

## 不适用场景

以下情况**不需要**调用本 skill：
- 环境完全确定且已知（直接设计最优方案即可）
- 没有在线反馈能力（没有观测就无法自适应）
- 决策必须一次性做出且不可逆转
- 试探成本高于执行错误成本（那就不试探，直接保守执行）
- 问题已经完全在模型内——先用 `constrained-optimization` 求解

## 方法论流程

### 第一步：诊断信息不完备的类型

第18章引言区分了三类不完备信息，诊断属于哪一种：

| 不完备类型 | 典型表现 | 核心挑战 |
|-----------|---------|---------|
| 输入信号不确定 | 市场需求波动、外部干扰 | 需要滤波和预测（先走 `state-estimation`） |
| 对象特性不确定 | 新领域、参数未知、模型不准 | 需要在线辨识和自适应 |
| 元件可靠性不确定 | 执行可能出错、组件可能失效 | 需要冗余和容错 |
| 环境参数漂移 | 条件缓慢变化使原方案渐失效 | 需要持续自校准 |

### 第二步：选择自适应策略

#### 策略 A：自寻优 / 极值搜索（第16章）

> 16.2 节：自寻最优点控制原理——加入试探信号，观测系统响应，根据响应方向调整控制，逐步逼近最优。

**适用条件**：
- 不知道系统的最优工作点在哪
- 可以施加小幅试探并观测响应
- 试探成本低于偏离最优的损失

**操作框架**：
```
循环：
  ① 施加小幅试探（改变输入 / 尝试新方法）
  ② 观测响应（性能指标的变化方向和幅度）
  ③ 如果变好 → 沿同方向继续移动
  ④ 如果变坏 → 反向或换另一个维度试探
  ⑤ 响应变小 → 说明接近最优点，减小试探步长
```

**类比到 agent 决策**：
- 面对一个新领域的策略选择，不要猜一个就固守
- 用小步试验探测策略空间的地形
- 每次只变一个维度，否则分不清是哪个因素造成了变化

**关键参数**：
- **搜索范围**：试探幅度多大？太大会错过峰值，太小会被噪声淹没
- **搜索周期**：多久试探一次？太快来不及观测响应，太慢跟不上变化
- **搜索损失**：试探本身会让性能偏离最优，这是为寻找最优付出的代价

#### 策略 B：自行镇定（第18.1-18.3节）

> 核心思想：如果系统可以在几种不同模式间切换，且其中至少有一种是稳定的——那么通过监测系统是否偏离可接受范围，一旦越界就随机切换到另一个模式，最终会"撞到"稳定模式并留在那里。

**操作框架**：
```
初始化：定义可接受边界（超出什么范围 = 失稳）
循环：
  ① 运行当前模式
  ② 监测是否触碰边界
  ③ 若触碰 → 切换模式（可以是随机的，也可以是有方向的）
  ④ 若在边界内稳定运行 → 保持当前模式
```

**类比到 agent 决策**：
- 面对一堆备选策略不确定哪个可行
- 设定明确的失败标准（什么情况下放弃当前策略）
- 一旦触发弃用标准，立即换一个策略试
- 如果某个策略一直没触发弃用标准，就继续用它

#### 策略 C：模型参考自适应（第18.6节）

> 核心思想：不直接知道"怎么做"，但知道"对的样子"——定义一个参考模型代表期望行为，实时比较实际表现与参考模型的差距，根据差距调整控制器参数。

**操作框架**：
```
① 定义参考模型：你期望的系统应该是什么行为？
   - 不一定是完整的数学模型
   - 可以是：期望的推进速度、期望的质量水平、期望的响应时间
② 并行运行：
   - 一边执行实际方案
   - 一边持续比较"实际输出 vs 参考模型输出"
③ 校正：
   - 偏差 e(t) = 实际 - 期望
   - 如果 e(t) 变大 → 调整执行参数使其缩小
   - 调整规则需要预先定义（哪些参数影响哪些输出）
```

**类比到 agent 决策**：
- 项目推进中，你知道"正常进度应该是一周完成X"
- 每周比较实际进度 vs 参考进度
- 偏差超过阈值 → 调整资源分配、方法、节奏
- 调整方向：如果慢了就加速，如果偏离就扶正

### 第三步：设计试探安全边界

所有自适应策略都涉及试探和切换，必须定义安全约束：

- **最大试探幅度**：不造成不可逆伤害的极限
- **回退条件**：什么情况下放弃试探回到已知安全方案
- **切换代价**：每次切换模式的成本是多少？如果切换太贵，适合更保守的策略

### 第四步：确定自适应速率

- **快速自适应**：环境变化快、试探成本低 → 大步试探、频繁更新
- **慢速自适应**：环境变化慢、试探成本高 → 小步试探、低频更新
- **自适应曲线**：初期信息少 → 快速试探（探索为主）；后期信息多 → 精细微调（利用为主）

## 黄金法则

1. **信息不完备时不追求一次做对**：设计一个能在运行中自我修正的系统
2. **试探和利用的平衡**：早期多试探（探索），后期多利用（精调）；但环境一旦变化要重新增强试探
3. **每次只变一个维度**：如果同时改变多个变量，无法归因哪个改变带来了改善
4. **失败是信息**：自行镇定中用随机切换找稳定模式——每一次"错误"的模式切换都在缩小搜索空间
5. **参考模型是锚**：在没有内部模型时，找到一个外部的期望标准作为参照，围绕它做校正

## 常见错误

| 错误 | 正确做法 |
|------|---------|
| 环境未知时固守一个固定策略 | 加入试探机制——在线学习需要"故意犯一些可控的小错" |
| 试探幅度太大导致系统振荡 | 接近最优时减小步长；斐波那契分段法可在最少试探次数内逼近极值 |
| 只观测不调整 | 自适应的核心是闭环——观测→判断→调整→再观测 |
| 把偶然波动当作系统变化 | 区分噪声和真正的参数漂移——用滤波先平滑观测再决策 |
| 没有定义"稳定"的标准 | 自行镇定必须先定义：什么范围内的状态是可接受的？什么不算失败？ |

## 与其他 skill 的关系

**认知位置**：修正环（主要）+ 执行环（辅助）。既是对偏差的响应机制，也是执行阶段的在线调节器。

**上游输入**：
- `investigation-first` + `systems-thinking-framework`：建立对系统的最初认知（哪怕不完整）——自适应需要初始模型，哪怕粗糙
- `state-estimation`：估计当前系统状态和不确定性，决定自适应速率
- `feedback-and-revision-loop`：检测到的偏差触发自适应调整

**同环协同**：
- `criticism-self-criticism`（审视思维过程）+ `adaptive-robust-strategy`（调整系统行为）= 人+系统双维修正

**下游交付**：
- 自适应调整后的方案 → `practice-cognition` 实践检验
- 长期运行的自校正 → `feedback-and-revision-loop` 持续监控

**特殊关系**：
- 与 `spark-prairie-fire` 深层共振——二者都在处理「信息不完备时如何前进」，前者是战略层面，本 skill 是战术实现
- 多子系统各自自适应可能互相干扰 → `hierarchical-decomposition-coordination`
